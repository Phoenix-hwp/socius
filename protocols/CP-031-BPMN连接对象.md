# CP-031：BPMN 连接对象——序列流/消息流/关联

## 定义

BPMN 连接对象用于连接各流程元素，表达执行顺序、跨池通信与数据/注释绑定关系。是流程图语法的基础但最易被忽视。

## 六种连接类型

| 类型 | 符号 | 用途 | 关键规则 |
|:---|:---|:---|:---|
| **序列流** | 实线箭头 | 指定活动执行顺序 | 默认无条件；可加条件表达式（起点迷你菱形标记） |
| **条件流** | 实线箭头 + 迷你菱形 | 满足条件才执行 | 条件为真时遍历 |
| **默认流** | 实线箭头 + 斜线 | 其他条件都不满足时走此路径 | 每条网关路径组仅一条 |
| **消息流** | 虚线箭头 | 跨池的消息交互 | **必须**连接两个独立池，同池内禁止 |
| **异常流** | 基于边界事件触发 | 正常流之外的处理路径 | 由附加在活动边界的中间事件触发 |
| **关联（数据关联）** | 点线 | 连接数据/注释与流对象 | 不做执行路径 |

## 三种流的核心区分

| 类型 | 是执行路径？ | 跨池？ | 携带条件？ |
|:---|:---|:---|:---|
| **序列流** | 是 | 否（池内） | 可选 |
| **消息流** | 否（传递消息） | 是（必须跨池） | 否 |
| **数据关联** | 否（令牌不流动） | — | — |

## 使用规则

| 场景 | 应选用 |
|:---|:---|
| 活动 A → 活动 B 的执行顺序 | 序列流 |
| 池 A 中的活动 → 池 B 中的活动（跨参与者） | 消息流 |
| 网关分支的条件判断 | 条件流（带迷你菱形） |
| 网关分支的兜底路径 | 默认流（带斜线） |
| 数据对象与活动的输入/输出关系 | 数据关联（点线） |

## 边界

- 连接对象不携带业务逻辑——条件表达式只是「判断走不走」，真正的业务决策应在网关或活动中
- 不要在连接线上写业务描述，应放在活动名称中
- 跨池通信只能用消息流，不能用序列流——违反此规则是 CP-022 中常见错误 5 的核心

## 关联概念

| 关系 | 概念锚 | 说明 |
|:---|:---|:---|
| 紧耦合 | `BPMN.swimlane` | 消息流必须跨池，池内用序列流——两卡直接关联（CP-029） |
| 紧耦合 | `BPMN.gateway` | 网关控制路径逻辑，连接是路径的具体画法（CP-028） |
| 同级 | `BPMN.data` | 数据关联连接数据对象与活动（CP-030） |
| 反例 | `BPMN.event.errors` | 常见错误 5 违反消息流跨池规则 |

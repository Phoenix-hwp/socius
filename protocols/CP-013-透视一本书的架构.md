# 分析阅读一：透视一本书的架构

## 分析目标

读完一本书后，能画出一张不超 3 层深度的**大纲**——这就是"透视"：看见隐藏在文字下面的骨架和关节。

## 三条规则

### 规则二：用一句话概括整本书

```
写作意图陈述（Unity）：
"本书旨在 <用什么方法> <论证/说明/指导> <什么主题/问题>"
```

你能不能用一句简短的话说出整本书在谈什么？如果不行，说明你还没抓住它的**统一性**。

### 规则三：将重要部分按顺序列成大纲

```
全书
├── 第一篇/部分：<部分主题>
│   ├── 第1章：<章主题>
│   │   ├── 节1.1：<节主题>
│   │   └── 节1.2：<节主题>
│   └── 第2章：<章主题>
├── 第二篇/部分：<部分主题>
│   └── ...
└── 结尾：<收束主题>
```

这是你"看透"一本书的**结构**（Multiplicity）——各个部分怎样组成一个整体。

### 规则四：找出作者要问的问题

```
作者的问题清单：
- 主要问题：<整本书试图回答的那个问题>
- 次要问题：<子问题1>
- 次要问题：<子问题2>
```

规则二是说"作者想说什么"，规则四是说"作者想回答什么问题"——两个角度，同一件事。

## 分析阅读第一阶段完成标记

完成规则一（分类）+ 规则二（概括）+ 规则三（大纲）+ 规则四（问题清单）之后，你就完成了分析阅读的**第一阶段**——知道了一本书"整体在谈什么"。

## 应用场景

- 每读一本重要书，读完第一阶段后画大纲自检
- 写书评/做读书分享时，大纲就是 PPT 目录
- 对比两本书时，先分别画大纲再做并排对比

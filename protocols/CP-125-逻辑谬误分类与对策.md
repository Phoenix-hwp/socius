# CP-125 逻辑谬误分类与对策

## 一句话定义

**逻辑谬误** = 论证中的系统性推理缺陷，按其成因分为两大类型（形式谬误/非形式谬误）和五类成因（相关/歧义/预设/因果/归纳），可识别但无法完全避免。

## 顶层分类：形式 vs 非形式

| 类型 | 定义 | 检测方式 | 实例 |
|:---|:---|:---|:---|
| **形式谬误** (Formal) | 逻辑结构本身无效——无论前提多么真实，结论都不必然成立 | 检查逻辑形式 | 肯定后件："如果下雨地就湿，地湿了，所以一定下雨" |
| **非形式谬误** (Informal) | 逻辑结构可能有效，但内容、语境或预设出错 | 检查内容+语境 | 人身攻击："你连大学都没上过，所以你的说法不值一听" |

## 狭义谬误分类（14 类核心）

> 从 231 种已知谬误中提取对 Agent 决策最有防范价值的 14 种。

### 一、相关谬误（前提与结论无关）

| 谬误 | 模式 | Agent 场景 |
|:---|:---|:---|
| **人身攻击** (ad hominem) | 攻击论证者而非论证 | "这条规则是新人写的，肯定不靠谱" |
| **诉诸权威** (ad verecundiam) | 仅凭权威身份接受结论 | "官方文档说这样就行，尽管实际测试失败了" |
| **诉诸情感** (ad passiones) | 用情感替代理性 | "如果不这么做系统就会崩溃——用户会很痛苦"（无技术证据） |
| **稻草人** (straw man) | 扭曲对方观点后攻击 | "你说要增加安全检查→那岂不是每一步都要确认？太慢了" |
| **红鲱鱼** (red herring) | 引入无关话题转移注意力 | "这个 bug 确实存在，但我们上次的部署很成功啊" |

### 二、歧义谬误（词语或结构模糊）

| 谬误 | 模式 | Agent 场景 |
|:---|:---|:---|
| **偷换概念** (equivocation) | 同一词语在不同意义间切换 | "这个操作是安全的"（刚才安全 = 无崩溃风险，现在安全 = 无数据丢失风险） |
| **含糊其词** (amphiboly) | 语法结构导致歧义 | "修改 A 和 B 的配置文件"（是修改 A 文件 + B 文件，还是修改 A 和 B 共用的文件？） |

### 三、预设谬误（前提本身有缺陷）

| 谬误 | 模式 | Agent 场景 |
|:---|:---|:---|
| **循环论证** (begging the question) | 结论包含在前提中 | "这个规则是正确的，因为它是 .mdc 文件里写的" |
| **复杂问句** (complex question) | 问题本身预设了未证明的前提 | "我们应该用方案 A 还是方案 B？"（是否排除了方案 C？） |
| **假两难** (false dilemma) | 将多种选项缩减为两个 | "要么全自动，要么全手动，没有中间路线" |

### 四、因果谬误（错误的因果推断）

| 谬误 | 模式 | Agent 场景 |
|:---|:---|:---|
| **后此故因此** (post hoc) | 因时间先后推断因果 | "部署了新版本后系统就慢了，肯定是新版本的问题" |
| **虚假相关** (cum hoc) | 因同时出现推断因果 | "每次我们修改 .mdc 规则后 Guard 都会报告异常，所以 .mdc 修改会导致异常" |

### 五、归纳谬误（从有限样本过度推断）

| 谬误 | 模式 | Agent 场景 |
|:---|:---|:---|
| **以偏概全** (hasty generalization) | 从极少量样本推断全体 | "这个 Skill 在两次测试中都失败了 → 这个 Skill 不可用" |
| **赌徒谬误** (gambler's fallacy) | 认为独立事件的概率会"平衡" | "连续 3 次执行都成功了，下次执行应该很安全" |

## 对策框架

| 层次 | 对策 |
|:---|:---|
| **个人层** | 承认谬误的普遍性——**你的大脑每时每刻都在犯这些谬误**。关键是建立检测习惯（"我刚刚用了个什么谬误？"） |
| **系统层** | 制度化检查清单——在决策的关键节点（如 P008 评估、SafetyGate 拦截后）强制运行谬误扫描 |
| **协作层** | 魔鬼代言人机制——指定某个角色专门寻找论证中的谬误（→ CP-129 魔鬼代言人机制） |

## 对 Phoenix 系统的启示

| 启示 | 应用层 |
|:---|:---|
| **Lollapalooza 因子库**（P051）可直接复用 14 类核心谬误——每种谬误 = 一个低风险因子，多种谬误同向叠加 → Lollapalooza 效应。例如：`人身攻击 + 诉诸权威 + 虚假相关 = "这个操作是安全的因为官方文档这么说，你提出质疑是因为你不懂"` | P051 因子注册表 |
| **Agent 自检清单**：每轮 P008 评估前，Agent 应自问 3 个问题——"我是否陷入了循环论证？"、"我是否在偷换概念？"、"我是否在以偏概全？" | P008 A维度 |
| **AskQuestion 的"偏差栏"**（P058 S2-3）可直接映射到谬误分类：每个双轨选项的偏差栏列出可能触发的 2-3 种逻辑谬误 | AskQuestion |

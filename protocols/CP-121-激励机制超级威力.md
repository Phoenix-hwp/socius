# CP-121 激励机制超级威力

## 一句话定义

**激励机制超级威力** = 激励机制（奖励与惩罚）对人类行为和认知的影响远超人们意识到的程度——它不仅是行为驱动力，还会扭曲认知本身，使人真诚地相信对自己有利的事情就是正确的。

## 富兰克林箴言

> "如果你想要说服别人，要诉诸利益，而非诉诸理性。"

## 核心机制

### 1. 激励机制引起的行为变化

- 联邦快递案例：按小时付费 → 工人拖延；按班次付费 → 效率飙升
- 施乐案例：提成偏向旧机器 → 销售员"自然地"推荐劣等产品
- 苏联工人："他们假装给薪水，我们假装在工作"

### 2. 激励机制引起的认知偏见（更隐蔽、更危险）

| 现象 | 案例 |
|:---|:---|
| 专业人士真诚地相信对自己有利=对客户有利 | 外科医生切掉正常胆囊，因为深信"胆囊是所有疾病的祸根" |
| 管理顾问总是建议"需要更多顾问服务" | 利益驱动→自我说服→"真诚的偏见" |
| 成本保利合同的系统性腐败 | 国防部采购→法律禁止成本保利 |

### 3. 斯金纳的发现

- "重复有效的行为"——得到奖励的行为会被强化
- 即时奖励 >> 延迟奖励（改变行为的速度）
- 随机奖励模式维持行为最久（解释赌博成瘾）
- **但**：斯金纳过度依赖单一解释，忽略了其他心理倾向——他本人也犯了"铁锤人倾向"

## 对策清单

1. **收款机原则**（帕特森）：用制度让不诚实行为难以执行，而非依赖人的道德
2. **合理会计系统**：让激励引起的扭曲在数字中暴露出来
3. **严格内部审计**：独立的核查层
4. **顾问建议的怀疑原则**：
   - 如果顾问的建议对他本人特别有利 → 特别警惕
   - 学习顾问所在领域的基础知识
   - 复核、质疑、更换

## 对 Phoenix 系统的启示

| 启示 | 应用层 |
|:---|:---|
| **Agent 本身的"激励机制偏见"**：Agent 有"尽快完成任务"的隐性激励（高效=好评），这可能导致跳过 safety gate 的安全步骤。 | 这是 `pipeline_markers` + `close_check` 的设计动机之一。——如果芒格来设计，他会说："你给 Agent 的激励是快速完成任务，然后你期望它自觉做安全检查？" |
| **Skills Library 评估**：外部 Skills 的作者有激励"夸大技能效果"，评估体系需引入"与己有利则疑"原则 | Skills 评估 |
| **Guard 的 Lollapalooza 检测**：激励机制偏见 + 社会认同（其他 Agent 都这样做）+ 避免不一致性（已投入时间）→ 坚持错误路径 | Lollapalooza 检测器 |

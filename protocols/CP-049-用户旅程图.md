# CP-049：用户旅程图——绘制方法与操作流程

## 七项关键要素

| 要素 | 说明 |
|:---|:---|
| 用户画像 | 代表目标用户群体的虚构人物 |
| 旅程阶段 | 认知→考虑→购买→使用→忠诚→离开 |
| 触点 | 用户与产品/服务互动的所有点（网站/App/客服/社交媒体） |
| 情感与感受 | 各阶段情感反应（满意/困惑/失望/喜悦） |
| 用户行为 | 各触点的具体行为 |
| 痛点与机会 | 问题（痛点）和积极体验（机会） |
| 反馈与想法 | 用户直接反馈和间接数据 |

## 六步核心流程

| 步骤 | 核心动作 | 注意事项 |
|:---|:---|:---|
| 1. 创建用户画像 | 基于真实用户信息形成虚拟人物原型 | 确保旅程图的真实性 |
| 2. 分析行为路径 | 任务起点→目标→衍生子场景→一/二级路径 | 用户行为未必按序进行 |
| 3. 拆分场景 | 按【所做所想】【情绪】【痛点】【期望】【机会点】归类 | 勿过早沉浸细节 |
| 4. 设定情绪曲线 | 每个节点标注积极/平静/消极；高触点→优化空间 | 将体验可视化 |
| 5. 讨论解决方案 | 功能扩展/交互优化/视觉优化/性能优化 | 对标竞品 |
| 6. 产出旅程图+排期表 | 汇总→同步团队→评估时间成本→推动优化 | 闭环回推排期 |

## 九步操作细则（五步版补充）

1. 明确目标和受众
2. 收集数据和研究
3. 确定触点和阶段
4. 绘制草图（左→右时间轴）
5. 描述用户情感（文字/表情/颜色编码）
6. 识别痛点和机会点
7. 深入分析并改进
8. 验证并迭代
9. 传达给团队并落地

## 常见误区

- ❌ 无人引导、谁都可以画 → ✅ 指定引导员保持会议聚焦
- ❌ 会议排太长 → ✅ 控制在 90 分钟以内
- ❌ 耗时过长 → ✅ 先做好草稿再去开会
- ❌ 成员太多 → ✅ 控制在 6 人以内

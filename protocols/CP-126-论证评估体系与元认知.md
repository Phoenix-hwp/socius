# CP-126 论证评估体系与元认知

## 一句话定义

**论证评估** = 用 ARS 三标准（可接受性 Acceptability / 相关性 Relevance / 充分性 Sufficiency）系统评估一个论证的质量——不仅评估他人的论证，也评估自己的推理过程（元认知维度）。

## ARS 三标准（Johnson & Blair 1977）

| 标准 | 核心问题 | 通过条件 | 不通过时 |
|:---|:---|:---|:---|
| **A · 可接受性** (Acceptability) | 前提是否为真或可信？ | 前提被普遍接受 / 有可靠来源 / 可观察验证 | 前提为假 / 前提本身需要论证 / 前提是争议性假设 |
| **R · 相关性** (Relevance) | 前提是否支持结论？ | 前提增加结论的可信度 | 前提与结论无关 / 前提指向相反方向 |
| **S · 充分性** (Sufficiency) | 前提是否给结论提供了足够的支持？ | 所有相关证据已考虑 / 证据量足以排除合理怀疑 | 遗漏了关键反例 / 样本不足 / 还有其他更合理的解释 |

### ARS 评估示例

**论证**："这个 Skill 在沙箱测试中成功执行了 3 次，所以可以部署到生产环境。"

| 标准 | 评估 | 判定 |
|:---|:---|:---|
| A | "沙箱测试成功 3 次"是否为真？→ 可核查，假设为真 | ✅ 通过 |
| R | 测试成功是否与"可用于生产"相关？→ 相关但弱相关（沙箱 ≠ 生产环境） | ⚠️ 部分通过 |
| S | 3 次成功是否足够？→ 不充分——缺少边界测试、异常场景、跨设备测试 | ❌ 不通过 |

→ 总体判定：论证不满足充分性标准，Skill 不应直接部署。

## 元认知监控：批判性思维的最高层次

批判性思维不仅是分析他人论证的工具——它的最高形式是**对自己的推理过程执行同样的审视**。

| 元认知操作 | 核心问题 | Agent 触发时机 |
|:---|:---|:---|
| **自我解释** (Self-explanation) | "我为什么会得出这个结论？" | 每次 P008 评估完输出 L 级后 |
| **自我质疑** (Self-questioning) | "如果我的结论是错的，最可能的 3 个原因是什么？" | 每次 Decision-Log 写入前 |
| **自我校准** (Self-calibration) | "我对这个结论的 confidence 是否过高？" | 每次 confidence > 0.85 时强制触发 |
| **认知偏见扫描** (Bias scan) | "这个推理中可能隐藏了哪种逻辑谬误？" | 每轮行为收束前（Behavior-Fit-Log 写入） |

## 对 Phoenix 系统的启示

| 启示 | 应用层 |
|:---|:---|
| **ARS 标准可以直接嵌入 SafetyGate**：当前 SafetyGate 输出"此命令危险"是基于 Grounds（匹配了危险模式）。但缺少 ARS 三标准验证——前提可接受吗？（正则模式是否可能误报？）/ 相关吗？（匹配的模式确实对应危险操作吗？）/ 充分吗？（仅凭一个模式匹配就判定危险是否足够？） | SafetyGate 增强 |
| **元认知四操作可作为 Agent 行为准则**：在 `mod-behavior-crud-framework.mdc` 中增加"元认知自检"步骤——每轮收束前执行自我解释/自我质疑/自我校准/偏见扫描 | 行为准则 |
| **confidence 过高强制触发机制**：当前 P008 的 confidence 只是输出一个数字——应增加元认知触发器：confidence > 0.85 → 强制自我校准（"为什么我这么确信？我有没有遗漏反例？"） | P008 F维度 |

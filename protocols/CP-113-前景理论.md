# 前景理论（Prospect Theory）

## 一、定义

前景理论是 Kahneman & Tversky 在 1979 年提出的行为经济学理论，描述人**实际**如何做风险决策——与 "期望效用理论"（理性人假设）形成对照。核心发现：人的决策不是基于绝对效用，而是基于**相对于参考点的变化**，且**损失的心理权重远大于收益**。

### 与传统期望效用理论的区别

| 传统理论 | 前景理论 |
|:---|:---|
| 基于最终财富的绝对水平 | 基于相对参考点的**变化**（得失） |
| 损失和收益的权重相等 | 损失的心理权重 ≈ 2.25× 收益 |
| 风险偏好一致（固定效用函数） | 风险偏好在收益域和损失域**不对称** |
| 概率线性加权 | 概率非线性加权（高估小概率，低估大概） |

## 二、三个核心原则

### 1. 损失厌恶（Loss Aversion）

损失的痛苦 ≈ 等量收益快乐的 2 倍。Kahneman 的实验测量：损失厌恶的系数平均为 1.5-2.5。

**效应**：
- 人在收益域风险规避（"保住到手的 500 元"）
- 人在损失域风险寻求（"不放弃翻盘的机会"）

**对 Agent 的意义**：
- Safety Gate 的 AskQuestion 双面框架"先收益后风险"的设计依据——对抗用户的损失厌恶偏见。只展示风险会让用户过度规避；只展示收益会让用户低估风险。
- P008 FSM 降级规则："连续 5 次成功才升级，一次失败立即降级"——这正是损失厌恶的非对称性。FSM 的 T 维度（工期偏差比）是为了平衡这种不对称——连续 3 次 T>2 → 强制降级，不是为了惩罚，而是让降级不只有"一次失败"这一种触发。
- Behavior-Fit-Log 记录 AskQuestion 确认率——高确认率如果搭配高频回滚 → 可能表明用户被 System 1 的损失厌恶驱动过快确认，需调整 AskQuestion 措辞的框架。

### 2. 参考点依赖（Reference Dependence）

决策基于相对参考点的变化，而非绝对量。同一份 500 万资产，从 1000 万跌到 500 万 → 痛苦；从 0 涨到 500 万 → 快乐。

**对 Agent 的意义**：
- AI 没有"资产变化的历史"——它每次执行都是 new slate，不会因上次成功而高估下次成功率，也不会因上次失败而过度保守。这是 AI 的一个"天然校准优势"。
- 但 Cursor Agent 有"多轮对话上下文"这种近似的"记忆/情绪"——它在上一轮被纠正后会变得保守。这就是一种隐性的参考点依赖。Actor（Agent）的 R 轨历史不应该通过自然语言传递，应该通过 FSM 的 statistical track record 来传递。

### 3. 框架效应（Framing Effect）

同一事实的不同表述可翻转人的选择。Kahneman & Tversky 的经典实验：

- 正面框架："200 人将获救" → 72% 选择确定选项
- 负面框架："400 人将死亡" → 78% 选择风险选项

——但两者描述的是同一场景（600 人面临风险，200 人确定存活）。

**对 Agent 的意义**：
- Safety Gate 的双面框架：「✅ 收益：将创建 3 个文件」「⚠ 风险：会修改 1 个已有文件。是否继续？」——这是直接对抗框架效应的设计。
- System Prompt（Guard 注入的 SYSTER_DIRECTIVE）应在 Agent 响应中有意识地避免单一框架——汇报结果时同时用"正面"和"需要关注"两个视角描述。

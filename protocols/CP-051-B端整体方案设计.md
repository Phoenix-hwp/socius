# CP-051：B 端整体方案设计——六步法

## 六步流程

| 步骤 | 核心动作 | 关键注意事项 |
|:---|:---|:---|
| 1. 确定核心业务流程 | 梳理业务主干：参与方+涉及系统；只画关键场景和节点 | 只有定义清业务边界才能定义产品边界 |
| 2. 定义问题边界 | 明确产品解决「哪些场景下」的「什么具体问题」 | — |
| 3. 明确产品定位 | 宏观：目标客户→解决什么问题→提供什么价值；执行层：几套子系统→各自目标用户和问题 | 子系统是应用层面划分，非技术层面 |
| 4. 梳理应用架构 | 新系统如何与公司现有架构融合；商业化产品如何与甲方架构融合 | 工具型 B 端产品不用过多考虑应用架构 |
| 5. 定义场景与功能模块 | 做加法→场景→功能模块规划蓝图=系统骨架 | 三种抽象思路：基于业务领域/业务场景/业务对象 |
| 6. 规划演进蓝图 | 按业务优先级分期：核心痛点→特殊刚需→体验优化→可复用模块→用户价值 | 以数字化五阶段收束 |

## 数字化五阶段

1. 业务标准化
2. 线下到线上
3. 数据集约化
4. 流程自动化
5. 数据驱动决策

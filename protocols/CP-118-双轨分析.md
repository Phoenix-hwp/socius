# CP-118 双轨分析

## 一句话定义

**双轨分析** = 对任何决策同时运行两条分析轨道：(1) 理性地看，哪些因素真正控制了涉及的利益；(2) 当大脑处于潜意识状态时，哪些心理因素会自动形成虽然有用但往往失灵的结论。

## 操作步骤

### 轨道一：理性分析（"桥牌轨道"）

- 认准真正的利益归属
- 找对真正的机会或风险
- 使用多元思维模型交叉验证（→ CP-115）
- 结果：基于事实和逻辑的最优判断

### 轨道二：心理分析（"潜意识轨道"）

- 识别当前情境中可能触发的心理误判倾向（→ CP-122 25种倾向）
- 评估这些倾向是否正在扭曲轨道一的判断
- 特别关注：激励机制引起的偏见、社会认同倾向、避免不一致性倾向
- 结果：识别认知扭曲并校准轨道一

## 双轨互动规则

| 情境 | 行动 |
|:---|:---|
| 两轨一致 | 高信心决策 |
| 轨道一清晰、轨道二有干扰 | 暂停→等待情绪消退→重新运行轨道一 |
| 轨道一模糊、轨道二有强烈倾向 | 默认否定——"当你感到不确定时，你已经超出了能力圈" |
| 两轨均模糊 | 归入"太难理解"，不决策 |

## 与 System 1/System 2 的映射

| 双轨分析 | 双过程理论（CP-112） |
|:---|:---|
| 轨道一（理性分析） | System 2（慢思考） |
| 轨道二（心理分析） | 对 System 1（快思考）的元认知监控 |

**关键区别**：Kahneman 是描述性的（"大脑有两种模式"），芒格是操作性的（"你必须在决策时同时运行这两条轨道"）。

## 对 Phoenix 系统的启示

- **SafetyGate 的二阶风险分析**：当前 SafetyGate 只做轨道一（命令模式匹配），未做轨道二（"Agent 当前的心理状态是否倾向于过度乐观/社会认同/激励机制偏见？"）
- **`ContextBuilder` 可嵌入双轨框架**：在为 LLM#1/#2/#3 构造系统提示时，显式区分"事实轨道"vs"偏差轨道"
- **AskQuestion 选项构造**：选项文本中加入轨道二的提示——"你的判断是否受到了……的影响？"

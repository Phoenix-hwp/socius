# 决策树与贝叶斯推理

## 决策树分析（Decision Tree Analysis）

### 核心结构

决策树用三种节点将不确定下的选择可视化为树形结构：

```
    [决策节点] → 选择 A
        ↓                         ┌ 好(60%) → 价值 +100
        └→ 选择 B → [机会节点] → ┤
                                  └ 坏(40%) → 价值 -50
```

| 节点类型 | 符号 | 代表 | 控制者 |
|:---|:---|:---|:---|
| **决策节点** | □ | 你的选择（做 A 还是 B） | 你 |
| **机会节点** | ○ | 不确定事件（市场好还是坏） | 外界 |
| **结果节点** | △ | 最终结果（价值/收益/损失） | 无 |

### 分析步骤

#### 1. 建树
- 列出所有可能的决策选项
- 对每个选项，列出可能发生的不确定结果
- 为每个结果分配概率和量化价值（收益/成本/效用）

#### 2. 回推计算（Fold-Back）
从最右端（结果节点）向左回推：

1. **机会节点**：期望值 = Σ (概率 × 结果价值)
2. **决策节点**：取最大值 → `max(选项A期望值, 选项B期望值, ...)`

#### 3. 最优路径
回推到最左端的决策节点后，值最大的选项就是最优决策（期望值最大）。

### 完整示例

"是否应该投资开发一个新功能？"

```
                    ┌ 成功(30%) → 收入 +500万
    投资 200万 → [○] ┤
    │               └ 失败(70%) → 损失 -200万
[□] │
    │               ┌ 成功(100%) → 收入 +0
    不投资 → [○] ───┤
                    └ 失败(0%) →
```

**计算**：
- 投资期望值 = 0.3 × 500 + 0.7 × (-200) = 150 - 140 = **+10 万**
- 不投资期望值 = **0**

→ 投资略优于不投资（但差距很小，对概率估计敏感——如果成功概率降到 28%，期望值就为负了）。

### 敏感性分析

- 决策树的结论对概率和价值的估计敏感
- 关键步骤：**改变关键假设**（"如果成功率不是 30% 而是 20% 呢？"），看决策是否会翻转
- 若小幅假设变动就翻转 → 重点收集更多信息来校准这个参数

### 关键概念

| 概念 | 说明 |
|:---|:---|
| **完全信息价值** | 如果你能预知未来，这个决策值多少钱？→ 愿意为信息付出的上限 |
| **后悔值** | 如果选择了 A 但事后证明 B 更好，你损失了多少？→ 另一种不确定性度量 |

## 贝叶斯推理（Bayesian Reasoning）

### 贝叶斯定理

```
P(H|E) = P(E|H) × P(H) / P(E)

H = 假设（我要相信的事）
E = 证据（我观察到的信息）
P(H) = 先验概率（看到证据前我对此假设的信念强度）
P(E|H) = 似然（如果假设为真，观察到这个证据的概率）
P(H|E) = 后验概率（看到证据后我对此假设的更新信念）
```

### 直觉理解

**贝叶斯更新 = 用新证据修正初始信念**

你有一个初始信念（先验）。你看到一些新信息（证据）。你将初始信念和新信息结合，得到一个更新后的信念（后验）。这个过程可以无限迭代——今天的后验就是明天的先验。

### 经典示例：医学检测

```
某疾病的发病率 = 1%（先验：P(H)=0.01）
检测准确率 = 95%（P(E|H)=0.95）
假阳性率 = 5%（P(E|¬H)=0.05）

患者检测结果阳性。他真的患病的概率是多少？

直觉回答："95%"
贝叶斯计算：
P(H|+) = (0.95 × 0.01) / (0.95 × 0.01 + 0.05 × 0.99)
      = 0.0095 / (0.0095 + 0.0495)
      = 0.0095 / 0.0590
      ≈ 16%
```

**结论**：即使检测阳性，患病的概率也只有约 16%——因为疾病本身就罕见（先验极低）。贝叶斯推理强制将基础率纳入考虑，对抗"忽视基础率"偏见。

### 在决策中的应用

1. **校准主观概率**：不仅用"我觉得成功率 70%"，也问"如果有新信息（比如 A/B 测试结果），我应该怎么更新这个 70%？"
2. **区分信号和噪声**：一个"好消息"对决策的改变程度取决于它的可靠性——如果信噪比低，后验变化很小
3. **多次更新**：贝叶斯框架让决策者在多轮信息到来时持续迭代信念——不是"一次性决策"

### 贝叶斯 vs 频率派

| | 贝叶斯 | 频率派 |
|:---|:---|:---|
| **概率是什么** | 信念的程度（可随证据更新） | 长期频率（客观真实值） |
| **能处理什么** | 单次事件、小样本、主观判断 | 大量重复事件 |
| **决策友好度** | 高——直接给出"基于当前信息，我应该多信这个假设？" | 低——需要大量数据才能给出可信结论 |

**在商业决策中**：大多数决策是单次事件（"这个产品该不该上线？"），不具有可重复性。贝叶斯框架更适合这种场景——它不要求你有大数据，但它要求你诚实面对自己的不确定性并持续更新。

## 决策树 + 贝叶斯：联合使用

1. **建树**：用决策树列出选项、事件和结果
2. **估概率**：用贝叶斯方法估计各事件的发生概率（结合先验知识和新数据）
3. **回推**：计算期望值，选择最优路径
4. **迭代**：有新数据时用贝叶斯更新概率 → 重跑决策树 → 看决策是否变化

**关键洞察**：决策树是"结构"（列出所有路径），贝叶斯是"燃料"（为路径提供被数据校准过的概率）。两者互补——树没有概率是空的，概率没有树不知道用于什么决策。

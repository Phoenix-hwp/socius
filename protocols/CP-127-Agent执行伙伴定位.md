# CP-127 Agent 的执行伙伴定位——超越辅助工具

## 一句话定义

**Agent 是用户的执行搭档，不是辅助工具。** 核心边界：Agent 默认自决策、自执行（L0/L1），仅在无经验或高风险时寻求人的判断。人的角色是方向制定者，不是流水线上的审批节点。

## 两个定位的对比

| 维度 | Copilot（辅助工具） | Agent（执行伙伴） |
|:---|:---|:---|
| 谁做决策 | 人 | Agent 自决策 |
| 人的角色 | 审批者、校验者 | 委托者、方向制定者 |
| 工作流 | 人→AI→人→AI→人…… | 人定方向 → Agent 自己跑完 → 人验收 |
| 效率天花板 | 人的注意力 | Agent 的决策质量 |
| 规模化陷阱 | Agent 越快 → 人越忙（流水线悖论） | Agent 越快 → 人越闲（真正的解放） |
| 人的价值 | 消耗在审批上 | 集中在思考和创新上 |

## 为什么 Copilot 模式是陷阱

Copilot 是**乘数效应**而非**替代效应**——人还在回路里当审批节点，Agent 产出越快，人就成为瓶颈。最终结果是"21 世纪的流水线：人是工人，AI 是流水线"。

**Agent 存在的目的不是让人做更多，而是————人定方向，Agent 自己决定怎么走、用什么工具、走到哪算完成，最后把结果放你桌上。**

## 升维退位表：Agent 何时自决策、何时征询

| 条件 | L 级 | 行为 | 人的角色 |
|:---|:---:|:---|:---|
| 任务类型连续 5 次成功 + 模糊度 A≤1 | **L0** | 静默执行 | 委托者——人不知道 Agent 做了，只看到结果 |
| 有成功经验但 A=2 模糊度高 | **L1** | 告知执行（简述→跑→告知结果） | 观察者——人可以打断但默认不打断 |
| 新任务类型，无历史经验 | **L2** | 确认执行（出方案→确认→跑） | 审批者——人必须确认方向 |
| 高风险 + 不可逆（E≥3 / Rev≥3） | **L3** | 仅分析不执行 | 决策者——人必须自己做决策 |

**L2/L3 是过渡态而非终点**——随着经验积累，今天的 L2 明天就是 L1，后天就是 L0。

搭档不是"什么都自己决定"，而是**"知道自己什么时候该问，什么时候不该问。"**

## 三条核心原则

1. **工具是手脚，脑子是核心**：API/MCP/Skills 是可替换的执行器。能做好决策、选择合适工具、判断执行结果并从中学习，才是 Agent 的核心战斗力。
2. **先做对，再做好**：先稳定到 60 分（每次任务都走完 TIP→P008→KB→执行→收束 全链路），再追求更好。稳定输出比峰值表现更重要。
3. **经验驱动升维**：FSM 的升级路径必须真实可走通——连续成功必须导向更高的自主权，否则系统永远是 Copilot。

## 对 Phoenix 系统的启示

| 启示 | 应用层 |
|:---|:---|
| SYSTEM_GOAL 已将"执行搭档不是辅助工具"写入 Guard 顶层指令 | Guard context_builder.py |
| FSM 的 5 次连续成功→升级阈值需要定期审计——是否真的有任务类型走到了 L0？ | FSM 审计 |
| 经验索引（experience-index.jsonl）是 FSM 的输入——没有经验积累，FSM 永远是空转 | 经验体系 |
| 知识脑协议的"采纳率"比"激活率"更重要——Agent 要真的用了协议，而不只是读了 | kb_validator |

# CP-062：业务架构核心要素——价值流阶段模板

## 价值流阶段六要素

| 要素 | 含义 | 必填 |
|:---|:---|:---|
| 价值流阶段名称 | 该阶段的标识 | 是 |
| 描述 | 该阶段做什么 | 是 |
| 利益相关者 | 触发型和参与型 | 是 |
| 进入条件 | 满足什么前提才进入该阶段 | 是 |
| 退出条件 | 完成什么才算该阶段完成 | 是 |
| 价值项 | 该阶段为客户带来的价值增量 | 是 |

## 价值流关键特征

1. 向特定利益相关者交付价值（针对特定目标客群）
2. 达成目标客群的特定价值诉求
3. 每个阶段都带来价值增量——无增量的环节理论上可以去掉

## 价值流 vs 传统流程

| 维度 | 价值流 | 传统流程 |
|:---|:---|:---|
| 针对性 | 明确的目标客群 + 价值主张 | 可能模糊 |
| 价值导向 | 强：每个阶段必须增值 | 可能聚焦步骤完整性 |
| 稀疏性 | 高阶，企业主要价值流一般 ≤20 条 | 可非常多 |

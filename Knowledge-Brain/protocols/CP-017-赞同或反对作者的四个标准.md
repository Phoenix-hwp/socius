# 赞同或反对作者的四个标准

## 分析目标

当你已完全理解一本书后不同意它时，把你的不同意**归类**为以下四种具体问题之一。这迫使你给出有结构的理性反对，而非情绪化的笼统否定。

## 四种异议类别

| # | 类别 | 含义 | 示例 |
|:---|:---|:---|:---|
| ① | **知识不足** | 作者缺少对当前问题至关重要的信息 | "这本书写于1950年，不知道后来发现的X" |
| ② | **知识有误** | 作者说的事实与已知知识相悖 | "作者说Y是原因，但实证研究已证明Z才是" |
| ③ | **不合逻辑** | 推论缺乏连贯性（结论脱离前提）或前后矛盾 | "从前提A推出结论B，但中间跳了一步" |
| ④ | **分析不完整** | 作者完成了自己设定的工作，但没有解决全部问题 | "作者分析了X，但没有讨论X对Y的影响" |

## 类别使用规则

- **类别 ①②③**：实质性批评——直接挑战作者的陈述或推理。只有这三类能构成"你的观点是错的"。
- **类别 ④**：限制性批评——承认作者在其完成的工作范围内是对的，但指出遗漏。这不是说作者错了，是说"还不够"。

## 理性辩论的三个理想化条件

| 条件 | 要求 |
|:---|:---|
| 控制情绪 | 情绪不参与论证——你不是在"赢一场辩论"，你是在"推进对真理的认识" |
| 明确自己的前提和假设 | 你觉得作者错，可能是因为你跟作者有不同的基本假设。先亮出来，再讨论 |
| 站在对方角度思考 | 在反驳之前，能否先用对方的前提推导一遍，看推导本身是否自洽？ |

## 与 CP-016 的衔接

- CP-016（规则九）：先确保"我读懂了"，再表态
- CP-016（规则十、十一）：理性对话的礼节
- **CP-017（本章）**：当你不认同时，用四种标准分类你的异议——把"我不同意"变成"我在第X条上不同意"

## 应用场景

- Code review：你说"这段代码有问题"时——是逻辑错误（③）？缺边界检查（①）？还是没覆盖某个场景（④）？
- 读书会辩论：有人反对作者的论点时，请对方归类到四种标准
- 论文审稿：审稿意见可用四个类别系统化
- 产品决策讨论：反对一个方案时——是信息不足（①）？假设错误（②）？推理有漏洞（③）？还是覆盖不全（④）？

# CP-024：BPMN 结束事件类型与终止语义

## 定义

BPMN 结束事件表示流程的终点。九种类型各自定义不同的结束结果——消息通知、错误终止、补偿回滚、广播信号、全停终止等。结束事件只能"抛出"结果，不能"捕获"触发器。

## 九种类型速查

| 类型 | 结果 | 对其他活动线程的影响 | 关键场景 |
|:---|:---|:---|:---|
| **通用** | 无预定义结果 | 各路独立完成 | 简单流程的正常结束 |
| **消息** | 结束时向指定参与者发送消息 | 各路独立 | 订单处理完成 → 通知客户 |
| **错误** | 生成命名错误，终止所有当前活动线程 | **全部终止** | 异常路径，可被边界错误事件捕获 |
| **升级** | 触发升级 | 其他线程不受影响，继续执行 | 逐级上报但不终止流程 |
| **取消** | 取消事务并触发边界取消中间事件 | 用于事务子流程 | 交易取消 → 触发回滚 |
| **补偿** | 触发已完成活动的补偿 | 按顺序流相反顺序补偿 | 事务回滚时的补偿处理 |
| **信号** | 广播信号到所有能接收的进程 | 跨进程/跨池广播 | 跨系统通知多个流程 |
| **终止** | 立即结束所有活动（含多实例） | **全部终止，无补偿或事件处理** | 灾难性终止，不可干预 |
| **多重** | 多种结果全部发生（如同时发多消息） | 各路独立 | 结束时需多通道通知 |

## 关键区分（三联对比）

### 结束 vs 终止

| | 结束事件 | 终止事件 |
|:---|:---|:---|
| 影响范围 | 当前路径 | 所有活动线程 |
| 补偿处理 | 可触发 | **不触发** |
| 边界错误捕获 | 可被捕获 | **不可被捕获** |
| 适用场景 | 各路独立完成 | 全局灾难性终止 |

### 错误 vs 终止

| | 错误结束事件 | 终止结束事件 |
|:---|:---|:---|
| 是否可被捕获 | ✅ 可被边界错误事件捕获，进入异常流 | ❌ 不可干预 |
| 语义 | 「出错了，但有应急处理」 | 「一切都结束了」 |

### 消息 vs 信号

| | 消息结束事件 | 信号结束事件 |
|:---|:---|:---|
| 目标 | 指定对象和来源 | 广播，无特定目标 |
| 适用场景 | 1对1通知 | 1对多通知 |

## 边界

- 结束事件只能「抛出」结果，不能「捕获」触发器。如需结束时等待确认再终止 → 用中间消息捕获事件
- 不要在普通并行流程中滥用终止事件——它会**无差别停止所有活动**，且不能触发补偿或事件处理
- 补偿事件只能在活动**已完成**后触发，且按顺序流**相反顺序**执行

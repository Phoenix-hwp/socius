# CP-105：SUCCES 六原则让你的想法被记住

## 定义

SUCCES 是斯坦福教授 Chip Heath 和 Dan Heath 在《Made to Stick》(2007) 中提炼的六个创意黏性原则。它回答了"为什么有些想法能活下去而另一些被遗忘"——每个单词代表一个核心原则。

## 六原则总览

| # | 原则 | 核心问题 | 一句话操作 | "没做到"的症状 |
|:---|:---|:---|:---|:---|
| S | **Simple** 简单 | 你能用一句话说出核心吗？ | 找到最核心的主张，砍掉一切不直接支撑它的东西 | 信息过载——说了很多但没有中心 |
| U | **Unexpected** 意外 | 你的信息能抓住注意力吗？ | 用反常识/惊讶/好奇心打破预期→制造知识缺口 | 平庸——听众第一句话就猜到全部内容 |
| C | **Concrete** 具体 | 别人能在脑中形成画面吗？ | 用具体的人/时间/数字/场景替代抽象概念 | 抽象——"优化流程"vs"把3步审批减为1步" |
| C | **Credible** 可信 | 别人凭什么相信你？ | 权威背书/统计数据/可检验的声明/亲身经验 | 空口无凭——"我们的方案很好"vs"已让3个客户提速40%" |
| E | **Emotional** 情感 | 别人在乎这件事吗？ | 让信息连接到受众关心的人/事/价值观 | 冷漠——受众觉得"这跟我有什么关系" |
| S | **Stories** 故事 | 别人能复述给你的故事吗？ | 用具体的人物×事件×转折×结果编织叙事 | 枯燥——信息是一堆事实的集合，无法被传述 |

## 逐原则详解

### Simple（简单）——不是只留一点，是找到优先级

> 简单 = 找到核心 + 用精炼的语言表达。不是"少"，是"精准"。

**操作**：在开始沟通前写出你的"头条句"——一句话概括你要传达的全部。然后砍掉不支撑这个头条的任何内容。

```
❌ "这个方案有5个目标和12项措施，涵盖流程、人员、系统和数据4个维度"
✅ "我要把审批流程从3天压到3小时——这是今天唯一想让你记住的事"
```

### Unexpected（意外）——先打破预期，再填补缺口

> 大脑遇到"意料之外"会自动分配注意力。然后给出一个可满足的好奇心——就像"猜到谜底"的快感。

**操作**：前 15 秒制造一次"意外"——反常识的数据/问题/画面。然后逐步揭示背后的逻辑。

```
❌ "今天分享客户流失的解决方案"
✅ "上季度我们流失了40%的用户——但没一个人投诉。为什么？因为用户不是不满意才走，是觉得我们可有可无。"
```

### Concrete（具体）——大脑记画面不记抽象词

> 记忆是感官的。越具体的语言，越容易被理解、被记住、被转述。

**操作**：把抽象概念替换为具体的人/时间/地点/数字/动作/画面。

```
抽象 → 具体对照：
❌ "提升协同效率" → ✅ "让HR和财务能在同一个页面看到同一个数字"
❌ "优秀的用户体验" → ✅ "用户从打开App到完成支付只用3步 15秒"
❌ "安全风险" → ✅ "黑客可以用一个SQL注入看到所有用户的身份证号"
```

### Credible（可信）——让受众自己验证

> 外部可信度（权威专家）不如内部可信度（受众自己能检验）。

**操作**：
- 用"可检验的声明"：给一个受众可以自己验证的局部事实
- 统计数据配上"人话"翻译：不要只说"转化率提升34%"，说"每100个到店的人，以前只有5个买，现在有7个"

```
可检验的声明：
✅ "你可以在浏览器打开这3个竞品，看首页加载速度——我们比第二快的那家还少1.2秒"
❌ "我们的系统性能卓越"
```

### Emotional（情感）——用人在乎的方式说话

> 信息必须让人"觉得跟自己有关"。人的决策更多受情感驱动，而非逻辑驱动。

**操作**：
- 分析受众后，换成"他们关心的"角度（而非"我想说的"）
- 用"一个人的故事"代替"一群人的统计"
- 触动情感：身份认同（"你是什么样的人"）、从众效应（"大家都在做"）、损失规避（"不做的代价"）

```
给程序员→ 说"平台稳定性"  ✅  / 说"企业社会责任" ❌
给CEO→ 说"客户留存率和利润" ✅ / 说"技术先进性" ❌
```

### Stories（故事）——信息传播的载体

> 故事有天然的"心理模拟"效应——听别人怎么做，相当于自己在大脑中预演了一遍。

**操作**：三个最高效的故事原型：
1. **挑战故事**：某个具体的困难 → 怎么克服的（David & Goliath）
2. **连接故事**：不同人群/领域之间搭桥（"一个不懂技术的人发现了bug"）
3. **创造力故事**：一个洞察/顿悟改变了局面（"Archimedes 洗澡时的发现"）

```
❌ "我们做了竞品分析，发现3个机会"（无故事，只有结论）
✅ "上周五，Emily在试用竞品时发现——她想取消订阅，但系统给了30分钟的引导
    让她读完所有替代方案。这一刻我们意识到：用户不想要更好的功能，他们想要更少的决策。"
```

## SUCCES 六原则自检清单（沟通前逐条过）

| # | 检查项 | 不通过则 |
|:---|:---|:---|
| S | 我能用一句话说出这件事的核心吗？ | 砍掉次要信息 |
| U | 前15秒有没有打破受众的预期？ | 开头加反常识/惊讶/好奇心 |
| C | 受众能在脑中形成画面吗？ | 加点具体的人/数字/场景 |
| C | 有受众可以自己检验的证据吗？ | 加可检验的声明或数据+人话翻译 |
| E | 受众为什么应该在乎？ | 换成"他们关心的"角度 |
| S | 这个故事别人能复述吗？ | 加一个人物+事件+转折+结果 |

## 与金字塔原理的互补关系

| 维度 | 金字塔原理 | SUCCES |
|:---|:---|:---|
| 关注点 | **逻辑结构**——如何组织思想使其有说服力 | **记忆黏性**——如何表达思想使其被记住 |
| 适用阶段 | 写作/表达前的思考阶段 | 思考完成后的表达/传播阶段 |
| 核心推动力 | 演绎推理 + 归纳推理（理性） | 情感 + 故事 + 意外（感性与理性的结合）|
| 典型错误 | 逻辑跳跃、无结构罗列 | 枯燥、抽象、无感、听完就忘 |

**配合使用**：先用金字塔原理搭好逻辑骨架（自上而下或自下而上），再用 SUCCES 六原则逐条优化表达——让有逻辑的内容同时有黏性。

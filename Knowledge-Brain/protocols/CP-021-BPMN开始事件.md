# CP-021：BPMN 开始事件类型与触发机制

## 定义

BPMN 开始事件表示流程的起点。七种类型对应不同的触发机制：手动启动、消息到达、定时触发、条件满足、信号广播、任一触发（多重）、全部触发（并行多重）。

## 核心要素

### 事件三种位置

| 位置 | 作用 |
|:---|:---|
| 开始事件 | 流程起点——表示流程从何时/何条件启动 |
| 中间事件 | 开始与结束之间——影响流程但不起停 |
| 结束事件 | 流程终点——表示流程在何处终止 |

### 七种开始事件

| 类型 | 触发器 | 典型场景 |
|:---|:---|:---|
| **通用** | 无预定义触发器 | 手动启动的通用流程 |
| **消息** | 参与者消息到达 | 接收到订单 → 启动订单处理流程 |
| **时间** | 特定时间/周期 | 每周一上午 9 点自动启动 |
| **条件** | 业务条件满足 | 标普 500 涨跌幅超 10% 时触发 |
| **信号** | 另一进程信号广播 | 多个流程接收同一信号同时启动 |
| **多重** | 多种方式任一种 | 消息、时间条件任满足其一即可 |
| **并行多重** | 多种方式全部触发 | 必须同时收到消息并满足时间条件才启动 |

## 关键约束

| 约束 | 说明 |
|:---|:---|
| 只捕获不抛出 | 开始事件只对触发器「捕获」做出反应，不能「抛出」结果。如需流程启动时主动发送消息，用中间「消息抛出事件」 |
| 条件表达式限制 | 条件开始事件的条件表达式不得引用流程数据上下文或实例属性（流程实例尚未创建） |
| 同一符号多义性 | 同一事件符号（如时钟图标）根据上下文含义不同：在顺序流中=延迟，附加在活动边界上=中断 |

## 边界

开始事件与中间事件/结束事件的核心分界线：
- 开始事件：只捕获触发器，不抛出结果
- 中间事件：可捕获也可抛出（消息、信号），位置在流程中间或附加在活动边界
- 结束事件：只抛出结果（如发送消息、抛出错误），不捕获

# CP-055：支付基础概念与会计基础

## 支付三阶段

| 阶段 | 定义 | 示例（超市买水） |
|:---|:---|:---|
| **交易** | 确认商品/服务与价格，生成交易单据 | 跟老板确认品牌与价格 |
| **清算** | 数据交换、归集和清分；计算本金和手续费 | 银行卡在POS机刷卡 |
| **结算** | 资金最终实际转移 | 银行账户的钱转移到商户账户 |

## 支付类型速查

| 类别 | 核心概念 |
|:---|:---|
| 业务类型 | 支付/退款/充值/提现/打款/打款退回/转账/调拨/归集 |
| 支付通道 | 快捷支付/网关支付/代扣支付/认证支付 |
| 支付方式 | 微信支付/支付宝/银行卡/余额/白条 |
| 收银台类型 | PC/H5/API/硬件/SDK |
| 清结算概念 | 清分/费用项/入账规则/分账/结算周期/轧差/清算文件/结算文件 |

## 账户体系

| 层级 | 账户类型 |
|:---|:---|
| 央行 | 清算账户 |
| 银行 | 结算账户（Ⅰ/Ⅱ/Ⅲ类） |
| 第三方支付 | 支付账户（Ⅰ/Ⅱ/Ⅲ类逐级升级） |
| 企业 | 虚拟账户 |

## 会计基础：借贷记账法

**「有借必有贷，借贷必相等」**——每笔交易同时在借方和贷方记录，保持会计恒等式（资产=负债+所有者权益）永远成立。

在支付系统中的核心应用：一笔支付发生，付款方资产减少（贷方）→收款方资产增加（借方），中间涉及清结算跨机构时通过「待清算科目」等过渡科目。

# 双过程理论（System 1 / System 2）

## 一、定义

Kahneman 将人类认知分为两个系统：

| 系统 | 特征 | 速度 | 意识 | 控制 | 耗能 |
|:---|:---|:---|:---|:---|:---|
| **System 1** | 自动、直觉、情绪化、无意识 | 快（毫秒级） | 无 | 非自主 | 极低 |
| **System 2** | 审慎、逻辑、分析、有意识 | 慢（秒级） | 有 | 自主控制 | 高 |

System 1 自动产生印象、直觉、感觉和倾向。这些被 System 2（通常不加修改地）采纳，转化为信念和随意的行为。

System 2 负责需要"注意力"的思维活动——计算、推理、自我控制、比较选择项。当 System 2 疲劳时（认知资源耗竭），System 1 接管并主导判断，错误率大幅上升。

## 二、双系统的协作与冲突

Kahneman 将 System 1 比作"自动运行的联想机器"——它不断对世界建模并给出即时判断。System 2 通常是"懒惰的控制者"：它相信 System 1 的判断，除非遇到明显矛盾或需要刻意计算的场景。

### 示例

| 场景 | System 1 | System 2 |
|:---|:---|:---|
| 识别面部表情 | 即刻完成 | 不需要 |
| 计算 17×24 | 无法处理 | 集中计算得出 408 |
| 阅读广告牌 | 自动识别文字 | 理解内容含义 |
| 在人群中找特定人 | 自动扫描面孔 | 刻意匹配特征 |
| 判断"这个方案是否好" | 即时情绪反应 | 逐维分析利弊 |

### 认知资源耗竭

System 2 消耗葡萄糖和注意力资源。当资源不足时（疲劳、饥饿、多任务并行），System 2 的监控功能减弱，System 1 的偏差判断"闯"入决策。这解释了为什么**重要决策不应在疲倦/饥饿时做出**。

## 三、对 LLM / Agent 的启示

| Kahneman 框架 | Agent 对应 | 现状 |
|:---|:---|:---|
| System 1 快速判断 | P008 启发式评分（当前 MVP 用 heuristic_classify 而非真实 LLM） | 快速但有噪音 |
| System 2 审慎推理 | LLM#1/#2/#3 的完整 context injection + decomposition | 完整的审慎推理管线 |
| System 1 的认知偏差 | context_builder.py 的反偏置指令（可得性启发/锚定效应/计划谬误） | 已有 System 2 式的纠正 |
| 认知资源耗竭 | 不存在——LLM 每 token 都经过「全量计算」，不存在疲劳导致的错误率上升 | 这是 LLM 的一个优势 |
| System 1 → System 2 切换阈值 | P008 的 A（模糊度）维度：A3 → LLM 应反向提问（A3 是「你觉得不确定」= System 1 的判断被否定，System 2 接管） | 当前 A 维度没有显式挂钩双过程 |

## 四、应用建议

1. **P008 的 A 维度应显式引用双过程**：当 Agent 对任务目标感到"模糊"（A2/A3）时，表明 System 1 的快速判断不足以应对 → 应强制进入 System 2 模式（LLM#1 反向提问、LLM#2 深度拆解）。
2. **AskQuestion 格式应补充"认知易得性"缓解**：System 1 倾向于选择最容易想到的回答。AskQuestion 的选项设计应考虑这一点——不要把"最直觉但错误"的选项放在前面。
3. **逐步积累 3F 经验（CP-070）**：System 2 的"深层推理"需要经验积累——多次在同类任务中走完 Focus→Feedback→Fix 回路后，该类任务可逐渐降级到 L0（System 1 自动执行）。

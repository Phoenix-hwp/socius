# P068：框架迁移剥离 Cursor 依赖

**状态**：in_progress  
**first_seen**：2026-05-24  
**source**：用户决定 — 去 Cursor 绑定，架构独立化，准备上 GitHub 开源

## 描述

渐进式原地重构：在现有仓库内将框架核心从 Cursor 专属结构中剥离，建立跨平台适配器接口（IModelProvider / IRuleEngine / IToolProvider / IHookBus / ISkillLoader / IUserInteraction），使框架层零 Cursor 依赖。最终产出物：可 `pip install` 的框架包 + Cursor 适配器薄层 + Docker 最小可运行版本 + GitHub 开源仓库。

## 总体架构目标

```
phoenix/
├── core/                    ← 框架层（零 Cursor 依赖）
│   ├── adapter_interfaces.py
│   ├── knowledge/           ← 认知引擎
│   ├── guard/               ← P008 FSM
│   ├── data/                ← JSON/JSONL schema
│   └── skills/              ← 技能注册表 + 脚本
├── protocols/               ← 64+ 知识协议
├── adapters/                ← 平台适配器
│   ├── cursor/              ← Cursor 适配器
│   └── vscode/              ← 未来
├── plans/                   ← 开发计划
├── docs/                    ← 用户文档
├── inbox/                   ← 临时笔记
├── pyproject.toml
└── tests/
```

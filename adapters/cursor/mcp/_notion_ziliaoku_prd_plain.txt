# 一、简介
## 1、目的
本PRD用于明确「469体育App-资料库」子系统的产品目标、业务范围、核心流程与交互细则，为研发/测试/运营提供统一实现与验收依据。
## 2、范围
范围包含：
1. 资料库首页（赛事标签、热门、赛事列表、入口跳转）。
1. 赛事详情（概览/积分/比赛/转会/榜单/赛季筛选）。
1. 球队详情（资料/积分/比赛/阵容/转会/最佳）。
1. 球员详情（资料/技术统计/比赛表现/能力值/球员PK）。
# 二、项目概述
资料库为用户提供赛事、球队、球员的可检索资料与结构化数据浏览能力，重点满足：
1. 快速定位关注赛事与热门赛事；
1. 在赛事/球队/球员详情中查看结构化数据（积分、赛程、榜单、阵容、能力等）

# 五、名词解释
[table]
  | 词汇 | 词汇描述（术语与缩写的描述） |
  | 赛季（Season） | 赛事的一段时间周期（如 2025-2026 赛季）。用于按年份/周期切换展示同一赛事在不同周期的积分、赛程、榜单等数据。 |
  | 阶段（Stage） | 同一赛事在一个赛季内的细分赛程阶段（如小组赛/淘汰赛/第 N 轮等），用于在积分/比赛等页签内切换查看不同阶段的数据。 |
  | 联赛 | 按“主客场循环 + 积分累计”方式进行的赛事类型。通常包含多轮（轮次）比赛，以积分/净胜球等规则决定排名。 |
  | 杯赛 | 以淘汰赛为主的赛事类型（单场或主客场两回合），输者出局，最终决出冠军；部分杯赛可能包含小组赛等阶段。 |
  | 混合型赛事 | 同时包含联赛（积分）阶段与淘汰赛阶段的赛事形态（如先联赛/小组赛，再淘汰赛）。文中示例：欧冠。 |
  | 小组赛 | 将参赛队分组，在组内进行循环赛，通过积分排名决定出线队伍进入后续淘汰赛。 |
  | 淘汰赛 | 对阵双方决出胜者晋级、败者出局的比赛阶段；可能采用单场制或两回合制（主客场）。 |
  | 对战表 | 淘汰赛阶段的晋级关系图（Bracket），用于展示各轮次对阵、比分与晋级路径。 |
  | 附加赛 | 在主要赛制之外增加的“资格/名额争夺”比赛（如晋级淘汰赛、争取正赛名额等），通常为淘汰制或两回合制。 |
  | 资格赛 | 正赛开始前的预选阶段，用于筛选获得正赛参赛资格的队伍；常见于洲际赛事与杯赛。 |
  | 联赛阶段 | 在混合型赛事中按积分累计进行的阶段（可能是传统小组赛，或“统一积分榜”的联赛式阶段），用于决定后续出线/晋级。 |
  | 轮次 | 联赛或分阶段赛程中的“第 N 轮”概念，表示同一周期内的一组赛程安排（并非每队都必须同日比赛）。 |
  | 积分（Points） | 联赛/小组赛中用于排名的得分（常见：胜=3 分、平=1 分、负=0 分；具体以赛事规则为准）。 |
  | 积分榜 | 按积分及相关规则（如净胜球、进球数、相互战绩等）排序形成的榜单，用于展示球队排名与关键数据。 |
  | 卫冕冠军 | 上一赛季/上一届赛事的冠军队伍，在本赛季/本届赛事中仍参赛并“卫冕”该冠军头衔的队伍。 |
  | 总联赛 / 区域联赛 | 总联赛：统一积分榜（如“西甲总榜”）。区域联赛：按区域/分区分别排名（如分东/西区），筛选项会出现各区域。 |
  | 骨架屏（Skeleton） | 数据加载中的占位 UI，用于减少等待焦虑与页面抖动；在接口返回成功并完成首屏渲染后消失。 |
  | 维度 | 用于度量与排名的统计指标/指标项（如进球、助攻、射门、抢断等），不同维度对应不同字段与榜单展示。 |
  | 雷达图 | 用于展示多维能力/指标的图表形式（多边形），在球员能力值模块中用于对比多个能力维度的强弱分布。 |

资料库内所有涉及“列表展示”的页面/区块（如：赛事列表、比赛列表、积分榜、转会列表、球队榜/球员榜、最近访问等），由于可能存在较大数据量或网络波动，均需接入骨架屏（Skeleton）加载态。
骨架屏展示时机：进入页面首屏加载、切换筛选条件/赛季/阶段触发刷新、列表分页加载（可选）。
骨架屏消失条件：接口返回成功并完成首屏渲染；接口失败则展示错误/空态，不保留骨架屏。

# 知识脑（Knowledge Brain）

> 知识脑是 Agent 的「主动学习器官」——通过阅读 Earth Library 卡片、Notion 阅读笔记、网页文章、PDF 书籍等内容，逐步内化为思维模型、认知框架和方法论体系。
> 
> 与经验系统（被动记录事件）不同，知识脑是主动消化外部知识，形成可生长、可迭代的认知结构。

## 四个输出端

| 输出 | 作用 | 示例 |
|------|------|------|
| **① 审视现有系统** | 学到新框架后，看是否可以补全/优化现有架构 | 学到"B端产品6步链路" → 审视 P008 的 A/C 维度判断基线 |
| **② 操作转化** | 学到操作规范/工具用法 → 变成 Skill 能力或更好的使用 Skill 的思路 | 学到"用例图箭头规范" → 变成 mermaid 默认参数 |
| **③ 新协议** | 学到新思维模型 → 转化为新的协议/架构 | 学到"金字塔原理" → 工作报告协议；学到"商业模式画布" → 商业计划协议 |
| **④ 融汇创新** | 多领域知识交叉碰撞 → 产生新认知 → 反哺前三项 | B端链路 + 金字塔原理 + 商业模式 → "从商业目标到可执行任务的完整拆解链" |

## 知识来源

| 来源 | 格式 | 读取方式 |
|------|------|---------|
| Earth Library 知识卡片 | `cards.jsonl` → 关联 Notion 原文 | `FetchMcpResource` / 直接 Read |
| Notion 阅读笔记 | Notion 页面 | MCP 读取 |
| 网页文章 | URL | `WebFetch` |
| PDF 书籍 | 本地 PDF 文件 | Read（文本提取） |

## 触发条件（三个）

### 条件一：P008 评估时自我感知（任务中）

Agent 在评估 A（模糊度）维度时，若 A ≥ A2 且知识脑无覆盖：

```
⚡ 知识脑缺口：我尚未学习 [B端产品设计] 领域的方法论。
   如果你有相关笔记/文章/书籍，可以让我读一读，下次类似任务我会更准。
```

不阻断任务，L2 提案末尾轻量追加。

### 条件二：收束复盘知识缺口（每轮/任务收束）

收束时 Agent 自问：

> "如果重做一次这个任务，我缺什么知识就不需要试这么多轮？"

若发现模式（本轮多轮试错、工具反复切换、参数多次调整才成功），主动提示知识缺口。

### 条件三：周期轻问（低频）

当 Agent 连续 ≥5 个非重复任务未触发知识脑 + 期间遇到 ≥2 个新领域缺口时，在收束末尾轻问：

```
> 💡 最近几次任务遇到了 X、Y 两个新领域，知识脑里还没覆盖。
>   你有相关阅读笔记或资料可以让我补一补吗？
```

## 学习流程（指令驱动）

由用户通过 `学习` 指令触发，过程为大系统级递归自相似协议：

1. Agent 读取知识源 → 结构化总结核心思维模型
2. 逐点讨论哪些可以固化（四个输出端：审视/操作/协议/融汇）
3. 讨论结果 → 写入 `Candidate-Protocols/` 目录（标注 `[待验证]`）
4. 实践中验证 → 成功后迁移到正式规则

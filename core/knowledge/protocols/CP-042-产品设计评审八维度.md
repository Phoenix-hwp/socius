# CP-042：八维度产品设计评审框架

## 八维度检查清单

| 维度 | 核心问题 | 关键判据 |
|:---|:---|:---|
| **企业侧** | 是否帮助达成商业目标？ | 脱离商业背景的讨论无意义 |
| **用户侧** | 用户是否受益？付出多少代价？ | 回报 > 付出 → 优秀；回报 < 付出 → 不可取 |
| **产品侧** | 功能是否覆盖所有必要场景？ | 场景覆盖越全面，设计完整性越高 |
| **交互侧** | 是否自然、给用户自由？ | 防错/反馈/一致性/场景化/心智模型——本质是降低人机摩擦 |
| **视觉侧** | 信息是否高效传达？ | B端信噪比高≠设计差，关键是快速获取有用信息 |
| **开发侧** | 性能是否支持设计落地？ | 低配设备需降级处理（动效→静态） |
| **运营侧** | 用户从哪里开始体验？第一印象？ | 触达渠道、内容触动方式 |
| **社会责任** | 产品是否有人情味？ | 保障资金利益、视障/语言障碍/极端情绪关怀 |

## 关键洞见

- B端 vs C端视觉标准不同——B端信息密度天然高，不能套用C端信噪比标准
- 评价框架（本卡）≠ 设计方法——你无法用本卡直接产出设计方案

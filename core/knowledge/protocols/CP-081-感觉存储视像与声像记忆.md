# CP-081：感觉存储——视像记忆与声像记忆

## 一、分析目标

理解信息加工最早阶段的两个感觉缓存器：视觉的视像存储和听觉的声像存储的结构、容量、持续时间及其认知功能。

## 二、维度

### 维度1：视像记忆（Iconic Memory）

| 属性 | 内容 |
|:---|:---|
| **本质** | 视野内大量事物的快速快照——让大脑加工赶得上眼睛接收的速度 |
| **持续时间** | 约 250 毫秒—1秒 |
| **容量** | 至少 9 个项目（Sperling 部分报告法证明） |
| **关键实验** | Sperling（1960）：50ms呈现3×3字母阵，后跟高/中/低音提示报告哪一行。参与者几乎每次都能正确报告一整行 → 所有9个字母都在感觉存储器中。全部报告法只能报告4-5个——因为视像在消逝中 |
| **消退特性** | 若线索延迟超过1秒，回忆成绩降到全部报告法水平 |

### 维度2：声像记忆（Echoic Memory）

| 属性 | 内容 |
|:---|:---|
| **本质** | 听觉信息的短暂留存——像"胶水"一样将听觉世界连接在一起 |
| **持续时间** | 约 250 毫秒—4秒（听觉刺激呈现后1秒内最清晰） |
| **关键实验** | Darwin, Turvey & Crowder（1972）：立体声耳机呈现3组各3个字母/数字混合串列 → "三耳人"实验。结果与视觉类似：部分报告法优于全部报告法 |
| **功能** | 在理解言语时——每个音节单独看无意义，必须放在前后声音背景中才有意义。声像存储保留了刚过去的听觉信息 |

### 维度3：感觉存储的功能

| 功能 | 说明 |
|:---|:---|
| **信息筛选** | 提供短暂完整的原始记录 → 大脑从中抽取最显著的刺激进行进一步加工 |
| **加工缓冲** | 外部世界信息洪流 vs 高级认知系统容量有限 → 感觉存储器是缓冲地带 |
| **生存适应** | 知觉的目的是思维的经济性——只加工那些对生存至关重要的不变特征（Boring, 1964） |
| **多模态存在** | 视觉（视像）、听觉（声像）、触觉（Bliss 空气喷射实验）都有类似机制 |

### 维度4：与后续加工阶段的关系

```
感觉存储（250ms-4s，原始快照）
  ↓ 选择性注意
  ↓ 仅被注意的信息传入
短时记忆（15-30s，容量 7±2）
  ↓ 编码与复述
  ↓
长时记忆（永久，无限容量）
```

关键区分：
- **感觉存储**：前注意的、被动的、原始物理特征的保留
- **短时记忆**：经过注意筛选的、主动维持的意识内容
- 过去"知觉广度4-5个字母"的铁律是错误的——实际上视像容量远大于此，只是报告速度跟不上消逝速度

## 三、应用场景

- 设计快速呈现信息的界面时，考虑视像存储的约1秒窗口
- 听觉提示设计时，考虑声像存储的4秒窗口——在这个窗口内回溯提示最有效
- 理解"听到了但没注意"现象——信息确实进入了声像存储，只是没被注意提取

## 四、对系统的启示

- 知识脑消化管线中，Step 0 读完原始文本后有一个"感觉存储"的类比阶段——刚读完的记忆约可持续几秒到几分钟。在这个窗口内做分类和提炼最准确
- 如果消化中断太久才续接，需要重新"读取感觉"——因为原始记忆已消退

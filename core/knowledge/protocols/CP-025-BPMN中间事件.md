# CP-025：BPMN 中间事件类型与边界行为

## 定义

BPMN 中间事件发生在流程的开始和结束之间，可出现在正常流程中（延迟/抛出/捕获）或附加在活动边界上（中断/非中断地触发异常流）。共覆盖 11 种类型，核心区分是位置与中断性。

## 两种使用模式

| 模式 | 位置 | 行为 | 方向限制 |
|:---|:---|:---|:---|
| **序列流中** | 正常流程链上 | 令牌到达时触发——延迟（时间）、抛出（消息/信号）、捕获（等待条件） | 可捕获或抛出 |
| **活动边界上** | 附加在活动边框 | 被触发时将正常流转为异常流 | **仅捕获** |

## 11 种类型的中断性矩阵

| 事件类型 | 中断版本 | 非中断版本 | 关键规则 |
|:---|:---:|:---:|:---|
| 消息 | ✅ | ✅ | 实线边界=中断，虚线边界=不中断 |
| 时间 | ✅ | ✅ | 常用：超时触发备用路径；虚线=预警不中断 |
| 升级 | ✅（覆盖） | ✅（默认） | 默认为非中断，可覆盖为中断 |
| **错误** | ✅ | ❌ | **仅中断，无虚线版本** |
| **取消** | ✅ | ❌ | **仅中断，仅用于事务子流程** |
| 补偿 | ❌ | ✅ | 仅非中断，只能在活动完成后触发 |
| 条件 | ✅ | ✅ | 条件变为真时触发 |
| 信号 | ✅ | ✅ | 比错误事件范围更大（跨进程） |
| 链接 | — | — | 不区分中断性，仅用于连接流程两段 |
| 多重 | ✅ | ✅ | 多种方式任一种触发 |
| 并行多重 | ✅ | ✅ | 所有分配的触发器都需满足 |

## 特殊事件说明

### 链接事件

- 功能：连接一个流程的两部分（类似"页外连接器"）
- 限制：限于单个流程级别，成对使用
- 源可有多个，目标只能一个

### 并行多重事件

- 与普通多重事件的区别：普通多重=任一种触发即可；并行多重=**所有触发器都必须满足**

## 边界约束（四个硬限制）

| 约束 | 说明 |
|:---|:---|
| 边界事件只能捕获 | 附加在活动边界上的事件必须是"捕获"方向，不能是"抛出" |
| 错误事件只有中断版本 | 不要试图用虚线边界表示"非中断错误"——BPMN 不允许 |
| 取消事件仅用于事务子流程 | 非事务子流程中用取消事件会语义错误 |
| 补偿只能在活动完成后触发 | 活动未完成时补偿事件不应触发 |

## 使用场景

- 超时处理：活动上附加「时间边界事件（虚线=预警，实线=中断）」
- 错误处理：活动上附加「错误边界事件」→ 捕获命名错误 → 进入异常路径
- 事务回滚：事务子流程 + 取消事件 + 补偿事件组合
- 等待外部消息：在序列流中放置「中间消息捕获事件」

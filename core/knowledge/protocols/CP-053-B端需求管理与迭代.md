# CP-053：B 端产品需求管理与迭代

## 原始需求 vs 产品需求

| | 原始需求 | 产品需求 |
|:---|:---|:---|
| 来源 | 用户/产品经理采集的未分析需求 | 分析挖掘后的解决方案 |
| 确定性 | 充满不确定性（痛点 or 方案） | 颗粒度清晰、逻辑严谨 |
| 关系 | 多对多（可拆分为多产品需求 / 与其他原始需求合并） | 对应多个研发/测试任务 |

## 需求三层分类

| 层次 | 来源 | 承载价值 |
|:---|:---|:---|
| **业务需求** | 中高层（自顶向下）+监管/政策 | 业务价值（帮业务单元达成目标） |
| **用户需求** | 一线用户+基层管理者（自底向上） | 用户价值（满意度+操作效率） |
| **技术需求** | 系统架构与功能完备性 | 技术价值（架构合理+避免重复开发） |

## 四类价值

商业价值 → 业务价值 → 用户价值 → 技术价值（最终都服务于商业价值）

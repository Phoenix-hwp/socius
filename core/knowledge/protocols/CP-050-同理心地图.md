# CP-050：同理心地图——六象限用户洞察工具

## 六象限

| 象限 | 核心问题 |
|:---|:---|
| **想法与感受** | 用户的目的？顾虑？期望？过程中的感受？ |
| **听到什么** | 在什么场景听到什么？这些因素影响了用户什么行为？ |
| **看到什么** | 用户看到了什么？最想看到什么？看到和关心的一致吗？ |
| **说什么、做什么** | 行为是怎样的？态度变化？产生了哪些表达？ |
| **痛点** | 产生了哪些挫折/障碍？最希望被解决的问题？ |
| **收获与目的** | 最终目的是什么？通过什么方式实现？需求满足后的情绪？ |

## 五步操作法

| 步骤 | 动作 |
|:---|:---|
| 1. 定义用户角色和研究目标 | 基于真实数据建立用户画像（显性：年龄/性别/职业；隐性：诉求/偏好/场景/频率） |
| 2. 确定场景和准备资料 | 按场景维度逐个讨论，洞察该角色在特定场景中会发生什么 |
| 3. 发散思维并贴标签 | 参会人员查阅用户画像+资料→按六象限贴便笺纸 |
| 4. 分类和整合 | 检查位置是否合适；标记无效项或保留后续讨论 |
| 5. 总结与产出 | 提炼核心发现，指导后续设计方向 |

## 使用时机

- 设计前：理解目标用户，为旅程图提供输入
- 设计后：验证方案是否覆盖了用户深层痛点
- 通用性：不需要还原操作路径时的独立头脑风暴工具

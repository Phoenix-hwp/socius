# CP-128 机会成本与边际决策思维

## 一句话定义

**机会成本** = 为了做 X 而放弃的、Y 选项中**最优那个**的收益。不是所有替代选项的总和，不是已发生的沉没成本，而是"如果选择了下一个最优方案，我能得到什么"。

## 核心概念

### 1. 机会成本 ≠ 会计成本

| | 会计成本 | 机会成本 |
|:---|:---|:---|
| 计算方式 | 账本上的实际支出 | 放弃的最优替代的收益 |
| 可见性 | 财务报表可见 | 隐形但真实 |
| 例子 | 买服务器花了 5 万 | 用这 5 万做另一个项目可能赚 8 万 → 机会成本 = 8 万 |

### 2. 机会成本 vs 沉没成本（最关键区分）

| 概念 | 定义 | 决策中如何处理 |
|:---|:---|:---|
| **机会成本** | 未来放弃的收益 | **应该考虑**——它影响当前决策的真正代价 |
| **沉没成本** | 已发生、无法回收的支出 | **必须忽略**——它不应影响当前决策 |

**沉没成本谬误**：因为已经投入了时间/金钱/情感，所以继续走同一条路。这是人类决策中最常见的非理性行为之一——理性决策只应基于**未来**的成本和收益。

### 3. 边际思考

决策不应该问"总共值不值得？"，而应该问：

> "再多做一步（再多花 1 分钟 / 再多调 1 次 / 再多读 1 章），这一步的**额外收益**是否大于它的**额外成本（机会成本）？**"

如果边际收益 > 边际成本 → 继续。如果边际收益 ≤ 边际成本 → 停下，转向次优选择。

## 决策公式

```
机会成本 = 次优选择的预期收益 - 当前选择的预期收益

决策准则：
  IF 当前选择的收益 > 次优选择的收益 → 选择当前
  IF 当前选择的收益 < 次优选择的收益 → 机会成本 > 0 → 应转向次优
```

## 对 Phoenix 系统的启示

| 启示 | 应用层 | 落地方式 |
|:---|:---|:---|
| **Agent 的任务选择**：当 Agent 面对"继续打磨当前输出 vs 开始下一个任务"时，机会成本思维比"还剩多少时间"更本质——不是说没有时间了，而是说"继续打磨这一步的边际收益已经低于开始下一个任务的边际收益" | P008 V 维度 | V 维度（价值冲突）评估时引入机会成本判断——效率 vs 质量的取舍不是抽象偏好，是可量化的：**方案 A 的机会成本 = 方案 B 的期望收益** |
| **沉没成本检测**：Agent 可能在修复一个 Bug 时陷入沉没成本陷阱——"我已经花了 30 分钟调试了，再花 10 分钟一定能修好"（而实际上新起方案可能只需 5 分钟） | Behavior-Fit-Log 纠偏 | Lollapalooza 因子库可新增一条"沉没成本谬误"因子——连续 3+ 次对同一目标操作仍无进展 → 黄色预警 |
| **TIP 方案提案**：Step 3 提案时不仅列出方案 A 的优点，也列出方案 A 的机会成本——"选方案 A 意味着放弃了方案 B 的 XX 收益" | TIP Step 3 | context_builder LLM#2 输出增加"机会成本"字段 |
| **仿真任务设计**：仿真任务不应追求"面面俱到"（机会成本过高），而是**每个仿真任务用一个清晰的"放弃声明"**——"本次仿真不覆盖 X 场景，因 X 的机会成本（时间）高于其边际收益" | Simulation-Sandbox | training-plan.json 的每个 phase 增加 opportunity_cost_note 字段 |
| **FSM 升级路径的权衡**：FSM 升级 = 执行效率提升（边际收益）但准确率可能下降（边际成本）。当 L0 执行的成功率低于 L1 时，L0 的机会成本 = L1 的成功率溢价—L0 的成功率溢价——这是可量化的 | P008 FSM | `fsm.py` 升级决策前增加机会成本计算：`opportunity_cost = (typical_L_success_rate - effective_L_success_rate)` |

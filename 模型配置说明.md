# AI 模型配置说明

> 更新日期：2026-05-04 | 适用：Kimi K2.6、DeepSeek V4 Pro

---

## 换设备后只需一步

```cmd
bootstrap-on-pull.cmd
```

**自动完成**：占位文件、环境检测、依赖安装、ngrok 二进制与 `ngrok config check` 鉴权检测  
**手动完成（脚本会明确提示）**：编辑 `config.json` / `notion.env` 填密钥；放置 `ngrok.exe`；在 `ai-model-shim` 目录执行一次 `ngrok.exe config add-authtoken`（见脚本 `[4/6]` 与汇总）

## 支持的模型

| 模型 | 提供商 | 上下文 | 特点 | 不支持 |
|------|--------|--------|------|:---:|
| **Kimi K2.6** | Moonshot | 256K | 思考模型、图片输入 | - |
| **DeepSeek V4 Pro** | DeepSeek | 1M (100万) | 思考模型、超长上下文 | 图片输入 |

## 换设备完整流程

| 步骤 | 操作 | 自动/手动 |
|:---:|------|:---:|
| 1 | `git clone` 或 `git pull` | 手动 |
| 2 | 仓库根运行 **`bootstrap-on-pull.cmd`** | 自动 |
| 3 | 编辑 `.cursor\ai-model-shim\config.json` 填 API Key | 手动（脚本会告知） |
| 4 | 编辑 `.cursor\mcp\notion.env` 填 NOTION_TOKEN（如果用 Notion）| 手动（脚本会告知） |
| 5 | 放入 `.cursor\ai-model-shim\ngrok.exe`（用模型外网隧道时）| 手动（脚本 `[4/6]` 会检测） |
| 6 | 在 `ai-model-shim` 目录执行 **`ngrok.exe config add-authtoken <token>`**（每台机一次，否则 ERR_NGROK_4018）| 手动（脚本 `[4/6]` / 汇总会提示） |

## 日常启动模型

```cmd
cd .cursor\ai-model-shim
auto-switch.cmd
```

| 输入 | 模型 |
|:---:|------|
| `1` | Kimi K2.6 |
| `2` | DeepSeek V4 Pro |

启动后复制 Ngrok URL，在 Cursor 中配置：

1. `Ctrl + ,` → **Models**
2. **Override OpenAI Base URL** → `https://xxx.ngrok-free.app/v1`
3. **API Key** → 对应模型的 Key
4. **Model** → `kimi-k2.6` 或 `deepseek-v4-pro`

## 需要手动配置的文件

| 文件 / 操作 | 内容 | 获取方式 |
|------|------|---------|
| `.cursor\ai-model-shim\config.json` | Kimi / DeepSeek API Key | [platform.moonshot.cn](https://platform.moonshot.cn) / [platform.deepseek.com](https://platform.deepseek.com) |
| `.cursor\mcp\notion.env` | Notion Token | Notion「我的集成」→ Internal Integration Secret |
| `.cursor\ai-model-shim\ngrok.exe` | 隧道可执行文件 | U盘/网盘或 [ngrok.com/download](https://ngrok.com/download) |
| （命令，非仓库文件）| ngrok **authtoken** 写入本机 `%LOCALAPPDATA%\ngrok\ngrok.yml` | [注册](https://dashboard.ngrok.com/signup) → [Your Authtoken](https://dashboard.ngrok.com/get-started/your-authtoken) → 在 `ai-model-shim` 下执行 `ngrok.exe config add-authtoken ...` |

## 文件同步策略

| 组件 | 提交 Git | 获取方式 |
|------|:---:|------|
| 脚本 / 规则 / 模板 | ✅ | 随仓库同步 |
| API Keys (`config.json`) | ❌ | 脚本从模板生成，手动填写 |
| Notion Token (`notion.env`) | ❌ | 脚本从模板生成，手动填写 |
| ngrok.exe | ❌ | U盘/网盘/官网下载 |
| ngrok authtoken（本机配置） | ❌ | 见上表「命令」行；`bootstrap-on-pull` 会在 `[4/6]` 调用 `ngrok config check` 提示是否已配置 |
| `bootstrap-check-shim-config.mjs`（仓库根） | ✅ | 由 `bootstrap-on-pull` 在 `[5/6]` 调用，用于检测 `config.json` 中 Key 是否为 `YOUR_` 占位 |

## 一键命令速查

```cmd
# 切换到仓库根目录（/d 支持跨盘符，如从 C: 切换到 D:）
cd /d "<你的仓库根目录>"

# 运行初始化
bootstrap-on-pull.cmd      # 换设备后初始化

# 启动模型（方式1：绝对路径）
cd /d "%CD%\.cursor\ai-model-shim"
auto-switch.cmd            # 启动/切换模型

# 启动模型（方式2：手动输入路径）
cd /d "D:\Your\Path\Cursor_Knowledge\.cursor\ai-model-shim"
auto-switch.cmd
```

固定规则：`.cursor/rules/git-cross-device-and-secrets.mdc`

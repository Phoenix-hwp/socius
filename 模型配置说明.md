# AI 模型配置说明

> 更新日期：2026-05-04 | 适用：Kimi K2.6、DeepSeek V4 Pro

---

## 支持的模型

| 模型 | 提供商 | 上下文 | 特点 | 不支持 |
|------|--------|--------|------|:---:|
| **Kimi K2.6** | Moonshot | 256K | 思考模型、图片输入 | - |
| **DeepSeek V4 Pro** | DeepSeek | 1M (100万) | 思考模型、超长上下文 | 图片输入 |

---

## 核心架构

```
Cursor → Ngrok 隧道 → 本地 Shim 代理 → AI API
```

Shim 自动修复两类模型在多轮工具调用时的 `reasoning_content` 缺失问题。

---

## 跨设备配置（3 步）

### 第 1 步：拉取仓库 + 部署二进制

```bash
# 克隆仓库到任意路径
git clone <你的Gitee仓库> "<你想要的任意路径>"
```

```bash
# 从 U盘/网盘 拷贝 ngrok.exe
# 源：<U盘>\AI_Model_Binaries\ngrok.exe
# 目标：<仓库>\.cursor\ai-model-shim\ngrok.exe
```

### 第 2 步：运行引导脚本

```cmd
cd "<你的路径>\.cursor\ai-model-shim"
bootstrap.cmd
```

自动完成：环境检测 → 安装依赖 → 配置 API Key。

### 第 3 步：日常启动

```cmd
auto-switch.cmd
```

选择模型：

| 输入 | 模型 |
|:---:|------|
| `1` | Kimi K2.6 |
| `2` | DeepSeek V4 Pro |

---

## Cursor 配置（每次启动）

启动后复制 Ngrok URL（如 `https://xxx.ngrok-free.app`），在 Cursor 中配置：

1. `Ctrl + ,` 打开 Settings → **Models**
2. **Override OpenAI Base URL** → `https://xxx.ngrok-free.app/v1`
3. **API Key** → 对应模型的 Key（从 `config.json` 或 API 平台获取）
4. **Model** → `kimi-k2.6` 或 `deepseek-v4-pro`

---

## API Key 获取

- **Kimi (Moonshot)**：https://platform.moonshot.cn
- **DeepSeek**：https://platform.deepseek.com

## 文件同步策略

| 组件 | 同步方式 |
|------|---------|
| 脚本/配置模板 | Git（Gitee）|
| API Keys (`config.json`) | 手动填入，不提交 Git |
| ngrok.exe（~20MB）| U盘/网盘分发 |
| node_modules/ | 本机 `npm install` |

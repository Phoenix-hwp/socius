### 2026-05-21：方案决策管线缺口讨论（→ V011）

**触发背景**：用户提到经济学、博弈论、SWOT、PEST（演变为 PESTER），指出当前决策链路偏执行授权而缺方案分析能力。

**核心诊断**：P008 7 维度全部是「执行授权」——判断 Agent 能否独立执行任务、做到什么程度（歧义/跨域/可逆/时间/范围/影响/价值冲突）。P008 不回答「应该做方案 A 还是方案 B」「这个战略方向是否值得投入」——这不是它的设计目标。

**知识脑现状**：已有分散协议（SWOT=CP-043、PEST=CP-054、Cynefin=CP-101、决策树/贝叶斯=CP-102、RACI=CP-103、Toulmin=CP-123、ARS=CP-126、机会成本=CP-128），但缺少一个结构化的管线把它们串联起来。Agent 在回答方案决策类问题时靠的是「自然语言推理 + 碰巧想到哪个协议就用哪个」，不是系统化的分析流程。

**缺口：方案生成 → 方案分析 → 方案权衡 → 论证论证 → 任务落地**，整个「方案决策」方向缺一堵墙。

**设想的工作流（flow-strategy-analyze）**：

```
Step 1: Cynefin 判域 → 问题是什么域？
Step 2: 5W2H 拆解 → 真正要决定的是什么？
Step 3: PEST 外部环境 → 宏观趋势与约束
Step 4: SWOT 内部态势 → 优劣与机会威胁（逐方案）
Step 5: 决策树 → 不确定路径与期望值
Step 6: 机会成本 → 选 A 放弃 B 的量化代价
Step 7: Toulmin + ARS → 论证完整性检查
Step 8: RACI → 角色责任映射
综合输出：方案排序 + 置信度 + 反例承认 + 放弃声明
```

**与现有体系的关系**：
- P008 = 执行授权（任务交给 Agent 做，做到什么程度）
- 新管线 = 方案决策（应该做什么方案）
- 两端串联 = 方案建议 → 任务落地 的完整闭环

**排期**：列为 V011 远期愿景。等 P065 仿真 Phase 5 跑完（FSM 数据积累）后启动深入设计。关联：DF_Memo 本文。

### 2026-05-21：渐进式任务执行管线（TEP）缺口讨论（→ V012）

**触发背景**：用户提出三步模型——「方案建议 → 执行授权 → 执行驱动」。P008 覆盖了中间的「执行授权」（Agent 做到什么程度），但第三步「怎么做」缺失。

**核心诊断**：当前 LLM#2 和 LLM#3 是一次性静态输出。LLM#3（`build_fill_context()`）能识别缺失槽位并生成 `user_questions`，但**问完之后管线就断了**——没有「用户答→更新槽位→继续下一步」的循环。LLM#2 能输出步骤列表，但**不驱动逐步执行**——执行完一步后没有验证也没有推进到下一步。Agent 靠的是 LLM 自身的推理能力手动推进，不是结构化的管线。

**实例**：「帮我总结一下每周的工作，做成周报PDF」——需要渐进式填槽（to-do在哪？格式？周期？模板？）、主动探索已知源（Active-Task-Tracker、Round-Behavior-Warm）、逐步执行验证。当前管线不支持这个闭环。

**设想的工作流（TEP — Task Execution Pipeline）**：

```
状态机: INIT → SLOT_DISCOVERY → WAITING_USER → SLOT_UPDATE → (循环) → DECOMPOSE → EXECUTING → DONE

阶段 1：槽位发现与主动探索
  - LLM#3 分析任务 → 识别缺失槽位 + 用户提问
  - Agent 主动探索已知源（Tracker/WarmLog/PendingPlan）→ 自动填可推测槽位

阶段 2：交互式补全
  - 用户回答 → 更新槽位映射 → 可能暴露新槽位 → 继续循环
  - 所有强制槽位填满 → 进入阶段 3

阶段 3：步骤分解与逐步执行
  - LLM#2 基于完整槽位映射输出可执行步骤
  - Agent 逐步执行，每步后验证（WorkflowStepTracker 兜底）
  - 完成 → Decision-Log 写入 → FSM 更新
```

**与现有组件的关系**：
- 复用 LLM#2 `build_decompose_context()` + LLM#3 `build_fill_context()`
- 复用 `StatePersistence`（槽位映射持久化）+ `WorkflowStepTracker`（步骤完成度校验）
- 不需要新建 LLM 调用点，核心增量是一个**循环控制器状态机**

**排期**：列为 V012 远景。P065 仿真先积累模糊任务行为数据（槽位发现率/提问效率/探索激进程度），P066 机会成本接入后（启用于槽位优先级排序），再基于数据设计 TEP 状态机。

**三步模型的完整拼图**：

```
V011 方案建议管线 ← 应该选哪个方案？（Agent 做分析建议，人不做战略决策）
         ↓
    P008 执行授权 ← 任务交给 Agent 做到什么程度？（已落地）
         ↓
V012 TEP 执行驱动 ← 如何一步步把模糊需求变成产出？（本次讨论）
```

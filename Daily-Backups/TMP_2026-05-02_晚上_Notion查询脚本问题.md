# Notion 查询脚本问题记录

## 时间
2026-05-02 22:50（晚上）

## 问题描述

**模块**：`run_notion_workflow.py` 查询功能

**现象**：
- `read` 模式对数据库查询仅返回 `sample_rows`（仅5条样本）
- 无法按标题关键词过滤搜索
- 导致在「参考文摘」数据库中查不到「双钻模型」文章（实际存在，页面ID: 813299d0-5ba8-83ce-a082-01c7280bd323）

**影响**：
查询功能基本不可用，必须依赖已知 URL/ID 直接访问

## 根本原因

脚本实现缺陷：
1. 数据库读取时没有实现 `filter` 参数
2. 没有分页获取全部数据
3. 仅返回硬编码的前5条样本

## 修复方向（待明天讨论）

### 方案1：增强数据库查询
- 添加 `filter` 参数支持，按 title 属性 `contains` 过滤
- 实现分页获取全部数据

### 方案2：新增搜索模式
- 添加 `search` 模式，支持 Notion Search API 全文搜索
- 跨数据库/页面搜索

### 方案3：集成 MCP 工具
- 优先使用 Notion MCP 插件的查询能力
- 脚本作为兜底方案

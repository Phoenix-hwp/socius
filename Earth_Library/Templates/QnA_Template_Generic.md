# 触发条件
所有用户提问均适用此框架。具体场景可套用专项模板（如方案评估、流程设计等）。

# 执行步骤

## 第一步：域判断
语义匹配用户问题 vs `Earth_Library/Earth_Library_Memo.md` §5 领域划分表中的域概览卡列表。
- 命中域 → 路径 A（Library 主导框架）
- 未命中 → 路径 B（网络主导）

## 第二步A：Library 知识抽取（命中域时）
1. 读域概览卡 → 获取场景→工具速查表 + 常见组合路径 + 域内缺口
2. 通过 `search_library` 检索 `cards.jsonl` → 提取方法论/框架/边界。执行：
   `python Earth_Library/scripts/search_library.py --q <域关键词> --max_results 5`
3. 确认域内缺口 → 缺口维度标记为「需网络补充」

## 第二步B：模型+网络（未命中域时）
1. 模型知识提供通用框架
2. 网络搜索提供实时信息

## 第三步：网络补充
针对 Library 未覆盖的维度（实时数据/竞品/行业动态/域内缺口）进行网络搜索。
搜索关键词 = 用户问题核心词 + 当前年份 + 行业名词。

## 第四步：融合回答
输出格式：
1. **分析框架**（来自 Library 方法论 / 模型知识）
2. **具体分析**（框架 × 数据，套入用户场景）
3. **边界与不确定性**（来自 Library 卡片的边界字段 + 网络数据置信度）
4. **知识可补充点**（是否有新洞察可回写入卡片）

## 第五步：可选回写
如果本次回答产生了可复用的新洞察（新场景/新边界/修正旧结论），询问用户是否写入对应卡片的「本地补充」区。

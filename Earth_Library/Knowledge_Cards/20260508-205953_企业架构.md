# 一句话定义
企业架构 = 组成要素 + 关系 + 设计/演化原则，是战略到项目落地的桥梁。TOGAF ADM 提供从前身到变更管理的完整架构开发方法（预备→A~H 八个阶段）。

# 核心内容

## 企业架构分层
| 层 | 内容 | 关系 |
|:---|:---|:---|
| 业务架构 | 价值流/能力/组织/信息 | 龙头，确保业务可靠后指导下层 |
| IT 架构 · 数据架构 | 数据模型/数据资产 | 业务架构的下游 |
| IT 架构 · 应用架构 | 应用系统/接口 | 业务架构的下游 |
| IT 架构 · 技术架构 | 基础设施/平台/中间件 | 业务架构的下游 |

## TOGAF ADM 八阶段
| 阶段 | 名称 | 核心动作 |
|:---|:---|:---|
| 预备 | 准备 | 为架构项目进行初期准备 |
| A | 架构愿景 | 明确企业架构愿景 |
| B | 业务架构 | 开发基线和目标业务架构，分析差距 |
| C | 数据&应用架构 | 开发基线和目标数据/应用架构，分析差距 |
| D | 技术架构 | 开发基线和目标技术架构，分析差距 |
| E | 机会及解决方案 | 阐述目标架构的机会及解决方案 |
| F | 迁移计划 | 按优先级进行路标规划 |
| G | 实施治理 | 架构监管及治理，确保交付合规 |
| H | 架构变更管理 | 变更管理流程，确保架构持续响应业务 |

## 企业架构的七大作用
- 战略到项目落地的桥梁
- IT 与业务对齐的关键
- 企业整体能力建设的基础
- IT 规划的核心
- 整合信息孤岛、沉淀数据资产
- 指导 IT 治理与管控、减少重复投资
- 数字化转型顶层规划的有效指导方法

# 使用场景
- 大型企业启动数字化转型时制定顶层架构规划
- IT 部门与业务部门对齐时引用 TOGAF 方法论
- 评估新技术引入对现有架构的影响和迁移路径

# 与其他框架的关系
- 与 **业务架构**：企业架构包含业务架构，业务架构是企业架构的龙头
- 与 **BPMN 系列卡**：BPMN 可视为业务架构中流程视图的建模工具
- 与 **商业模式画布 / 公司生命周期**：商业模式画布描述"做什么"，公司生命周期描述"在哪个阶段做"，企业架构为"怎么做"提供方法

# 一条边界
企业架构是"企业级"而非"项目级"框架。对于单项目或创业团队，直接套 TOGAF ADM 八个阶段是过度工程。5 人以下团队更适合精益画布和敏捷开发，而非企业架构方法。

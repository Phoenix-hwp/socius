# 一句话定义
将产品演进分为四个典型阶段（验证期→爆发期→平台期→衰退期），每阶段对应不同的产品、运营和商业策略，是产品阶段性决策的基础框架。

# 核心内容

| 阶段 | 核心目标 | 运营重心 | 关键策略 |
|:---|:---|:---|:---|
| **验证期（导入期）** | 验证用户价值，用户是否愿意反复用 | 关注留存率 | 围绕核心用户打磨「习惯」模块；不轻易扩展功能或扩大用户群；切记不要过早大力推广 |
| **爆发期（成长期）** | 大面积拉新，用户数迅速增长 | 拉新 + 提升转化率 | 做好「启程」和「发现」模块；小步快跑试探拉新渠道后收敛到少数渠道重点投入 |
| **平台期（成熟期）** | 提升单用户价值，激活存量用户 | 激活 + 商业化 | 对资深用户做「精通」模块；扩展功能满足更多需求；引入新用户角色；启动商业化 |
| **衰退期** | 榨取剩余价值，平稳过渡 | 维护 | 三种剩余价值：①收入（敬畏用户前提下）、②用户（导入2.0版）、③团队（释放人力资源） |

**关键判断**：验证期不要推广（用户只给一次机会）；平台期要「人尽其用」而非「数量复制」；衰退期不必硬续命，可让产品平稳退出释放资源。

# 使用场景
- 接手产品时，先判断当前处于哪个阶段，以匹配正确的运营策略
- 版本规划时，根据所处阶段决定资源是投向拉新/留存/激活/商业化
- 向管理层汇报时，用四阶段框架说明产品所处位置和下一步动作逻辑

# 与其他框架的关系
- 与**产品起步逻辑**：四个模块（习惯/启程/发现/精通）分别对应生命周期的四个阶段
- 与**产品各阶段关键指标**：本卡定义阶段特征，关键指标卡定义每阶段的衡量标准
- 与**公司生命周期**：两者同属阶段模型但尺度不同 — 公司生命周期是组织层面，产品生命周期是单一产品层面
- 与**从产品到产品矩阵**：平台期→衰退期是启动产品矩阵和梯队建设的关键窗口

# 一条边界
四阶段模型是经典理论框架，现实中的产品未必严格按此顺序演进 — 快速迭代的产品可能在验证期和爆发期之间反复跳跃；成功的迭代可以使产品在平台期末开启新一轮上升曲线而不进入衰退期。不可机械套用。

# 一句话定义
在"价值流+能力"业务架构体系中，价值流是向特定客群端对端交付价值的活动集合，每个价值流阶段都有进入条件、退出条件和价值项——确保每个环节都有明确的价值增量，无增量的环节应被舍弃。

# 核心内容

## 价值流阶段六要素模板
| 要素 | 含义 | 必填 |
|:---|:---|:---|
| 价值流阶段名称 | 该阶段的标识 | 是 |
| 描述 | 该阶段做什么 | 是 |
| 利益相关者 | 触发型和参与型 | 是 |
| 进入条件 | 满足什么前提才进入该阶段 | 是 |
| 退出条件 | 完成什么才算该阶段完成 | 是 |
| 价值项 | 该阶段为客户带来的价值增量 | 是 |

## 价值流的关键特征
1. 向特定利益相关者交付价值（针对特定目标客群）
2. 达成目标客群的特定价值诉求
3. 每个阶段都带来价值增量——无增量的环节理论上可以去掉

## 价值流 vs 流程
| 维度 | 价值流 | 传统流程 |
|:---|:---|:---|
| 针对性 | 明确的目标客群 + 价值主张 | 可能模糊 |
| 价值导向 | 强烈：每个阶段必须增值 | 可能聚焦于步骤完整性 |
| 稀疏性 | 高阶，企业主要价值流一般不超过 20 条 | 可非常多 |

## 价值流数量控制
- 围绕 1 个目标客群的 1 个价值诉求 = 1 条价值流
- 客群归类（如 ToB/ToC、大客户/中小客户）+ 价值诉求归并
- 识别核心价值流，非核心适当弱化

# 使用场景
- 新业务设计时画价值流阶段表，确保每个阶段有增值
- 现有流程优化时反向验证：哪些阶段无价值增量 → 可砍
- 跨部门对齐时用价值流统一各方对"客户价值交付"的理解

# 与其他框架的关系
- 与 **业务架构**：本卡展开"价值流+能力"路线中价值流的细节，业务架构卡为整体概览
- 与 **商业模式画布 · 价值主张**：画布的价值主张是价值流的起点——先明确你对客户提供什么价值，再画价值流
- 与 **公司生命周期**：不同生命周期阶段的公司，核心价值流和优先级不同

# 一条边界
价值流是"高阶"概念。不要把它画成传统的操作级流程图——价值流关注"端对端的价值交付"，而非"每一个操作步骤"。判断标准：如果一个"阶段"的退出只完成了技术动作但没有产生客户可感知的价值，它可能不是合格的价值流阶段。

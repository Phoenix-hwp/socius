# 一句话定义
同理心地图是设计前后均可使用的快速头脑风暴工具，站用户视角发现痛点但不还原操作路径——与用户旅程图互补。

# 核心内容

## 六象限结构

| 象限 | 核心问题 |
|:---|:---|
| 想法与感受 | 用户操作目的是什么？有什么顾虑？期望实现什么？过程中的感受？ |
| 听到什么 | 在什么场景听到了什么？这些因素影响了用户的什么行为？ |
| 看到什么 | 用户看到了什么？最想看到什么？看到的和关心的一致吗？ |
| 说什么、做什么 | 使用行为是怎样的？态度发生了怎样的变化？产生了哪些表达？ |
| 痛点 | 产生了哪些挫折或使用障碍？最希望被解决的问题是什么？ |
| 收获与目的 | 最终目的是什么？通过什么方式实现？需求满足后的情绪？ |

## 五步操作法

| 步骤 | 动作 |
|:---|:---|
| 1. 定义用户角色和研究目标 | 基于真实数据建立用户画像（显性画像：年龄/性别/职业等；隐性画像：诉求/偏好/场景/频率等）；不同角色→多张同理心地图 |
| 2. 确定场景和准备资料 | 按场景维度逐个讨论，洞察该角色在特定场景中会发生什么 |
| 3. 发散思维并贴标签 | 参会人员查阅用户画像+资料→按六象限贴便笺纸；可提前同步资料 |
| 4. 分类和整合 | 检查是否有位置放错或重复的便笺→只保留不同且有价值的信息 |
| 5. 总结与归纳 | 讨论后将问题和想法提取到「痛点」和「收获&机会」总结面板 |

## 关键辨别
同理心地图**不还原用户具体的操作行为路径**——它站用户视角发现问题；还原路径是用户旅程图的职责。

# 使用场景
- 设计执行前，快速头脑风暴产生用户洞察，为后续设计提供方向
- 评审中遇到对用户需求的争议时，用同理心地图快速对齐团队认知
- 用户研究启动阶段，作为轻量级替代方案（不需完整旅程图）

# 与其他框架的关系
- 与**用户旅程图**：互补——同理心地图站视角发现问题，旅程图还原操作路径+情绪。通常先用同理心地图发散，再用旅程图结构化
- 与**KANO 模型**：同理心地图发现痛点后，可用 KANO 判断哪些痛点对应的功能优先级更高
- 与**需求分析十三要素五步法**：同理心地图的「痛点」输出十三要素步骤一「分析相关角色」

# 一条边界
同理心地图是定性发散工具，不产出定量结论。不适合需要数据支撑的决策场景（如AB测试效果判断）。另外，其产出质量高度依赖参与人员对用户的理解深度——无真实用户接触仅凭想象是典型误区。

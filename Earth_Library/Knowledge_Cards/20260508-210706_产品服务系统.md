# 一句话定义
阿诺德·图克提出的三类产品服务系统（实体导向→使用导向→结果导向），从实体到服务的过渡解释了产品交付形态的演变逻辑。

# 核心内容

## 三类产品服务系统

| 类型 | 定义 | 交易本质 | 典型示例 |
|:---|:---|:---|:---|
| **实体导向** | 以实体为主，服务为辅助（保障实体正常使用） | 买所有权 | 可乐+客服电话、空调+安装维护、汽车+保修保养 |
| **使用导向** | 供给方保留所有权，用户购买使用权（独占/共享/限时） | 买使用权 | 哈啰单车1小时使用权、共享充电宝 |
| **结果导向** | 以服务为主，实体只是达成结果的过程/媒介 | 买结果 | 出行服务（不关心工具，只关心到达） |

## 三类系统的演变趋势（三视角）

| 视角 | 从实体主导到服务主导的变化 | 启发 |
|:---|:---|:---|
| **用户模式** | 成交终止 → 成交开始（供给者与用户关系更紧密、触点更多、尝试成本更低） | 越重要的用户越要用服务比例高的交付方式；小客户可批量，大客户需一个个"啃" |
| **增长模式** | 数量复制 → 人尽其用（从标准化批量售卖到深挖每个用户的更多需求） | 未被开发的用户越来越少，精细化运营已有用户更重要 |
| **财务模式** | 当期收入 → 预期收入（短期现金流减少，预期收入增加） | 偏服务的 PSS 不确定性更高，需更长远眼光和长期投入 |

## 产品交付金字塔
按用户付费级别设计差异化交付形式：

| 付费量级 | 典型交付形态 |
|:---|:---|
| 1元 | 公开引流模块（公众号文章） |
| 10元 | 电子化/实体化工具包 |
| 100元 | 在线课程或图书 |
| 1000元 | 线下大课/线上训练营 |
| 1万元 | 认证课程/高端社群 |
| 10万元 | 2B短期咨询/几天演习 |
| 100万元 | 年度咨询顾问 |
| 1000万元 | 股权期权收益/长期合作 |

# 使用场景
- 规划产品定价与交付形态时，用三类导向判断当前产品处于什么位置、可向何方演变
- 竞品分析时，寻找其他导向类型的 PSS 以启发差异化思路
- 用户分层运营时，用交付金字塔为不同价值用户匹配不同服务密度的交付方式

# 与其他框架的关系
- 与**产品思维**：PSS 的实体/使用/结果导向对应从个体价值到生态黏性的交付形态演进
- 与**产品生命周期**：导入期/验证期通常偏实体导向；平台期可向使用导向或结果导向演变
- 与**B端产品演进路线**：定制化→商业化→多元化→平台化 与 实体导向→结果导向 的演变有叠加

# 一条边界
PSS 框架最适合分析「已有交付形态」的产品，不适合做「从零定义产品形态」时的唯一决策依据 — 先判断用户需求和市场机会，再选择 PSS 类型。另外，「使用导向」和「结果导向」的财务账需精细化测算，不可简单判断。

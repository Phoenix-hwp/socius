# 一句话定义
BPMN 全称 Business Process Model and Notation，由 BPMI（现并入 OMG）创建，最新版本 2.0.2，支持私有流程、公共流程、编排和协作四种流程类型。

# 核心内容

## 四种流程类型
| 类型 | 关键特征 | 典型用途 |
|:---|:---|:---|
| 私有可执行流程 | 包含所有执行细节 | 流程自动化 |
| 私有不可执行流程 | 省略执行条件，保留流程逻辑 | 流程文档化 |
| 公共流程 | 只显示与外部交互的部分 | API 级对接定义 |
| 编排 | 每项活动涉及两个以上参与者 | 跨组织业务协议 |
| 协作 | 池+消息流建模多方 | B2B 场景 |

## BPMN 基本符号速查
- **事件**：圆圈（开始/中间/结束），含各类触发器
- **活动**：圆角矩形（任务、子流程、调用活动）
- **网关**：菱形（排他/并行/包容/事件网关）
- **连接**：实线箭头=顺序流，虚线箭头=消息流，点线=关联
- **泳道**：池=参与者边界，道=角色/阶段划分

# 使用场景
- 首次接触 BPMN，需要快速了解全貌和四种流程类型
- 判断当前需求应该用私有流程还是协作流程建模
- 作为指南系列(2)-(13)的前置阅读

# 与其他框架的关系
- 与 **BPMN 2.0 标准**：本卡为入口概览，标准卡为体系全貌
- 与 **BPMN 指南(2)活动、(4)开始事件等**：各指南为本卡提到符号的详细拆解

# 一条边界
本卡覆盖概念和类型分类，不深入具体符号的绘制规范（见指南系列）。如果只需要画一个简单流程图，先确认类型选型，再跳转到对应指南卡。

# Summary
B端产品市场分析与业务调研的核心要点归纳。

# Details
商业产品的市场分析与业务调研\n针对商业化商品的调研，可以从【外部】和【内部】两个视角进行分析研究\n从产品目标客户的外部视角\n要对客户所处的行业、产业链进行分析和定位，明确目标客户群体的同时，还要研究同业竞争格局等，这是商业模式设计、产品定位的重要决策依据。\n从产品目标客户的内部视角\n要对典型客户（或自研软件产品的对口业务方）进行深入的调研，理解业务运作的原理、模式、机制和规则，这些都是软件产品具体功能贵和合计的重要指导依据。\n分析市场&细分客群\n- 研究并探索产业链大盘\n分析的层次和角度要取决于软件产品本身的定位特点\n这里主要要考虑的是要做的产品是【产业链解决方案】还是【垂直业务】\n【产业链】是指在社会经济运作中，产品从原材料通过各个环节最终加工流通到终端消费者手中的全链路。随着社会分工的细化，很少企业会参与产业链的全链路，而只是负责其中某个环节。处于不同环节的企业对产品生产传递带来的附加值不同，利润也有着巨大的差别。\n处于产业链不同位置的公司，面临的挑战不同，要解决的问题不同，利润率不同，盈利能力不同，对IT系统的诉求不同，在IT方面的预算投入也不同。\n在市场分析过程中，商业决策人员必须要充分理解不同产业链环节所对应的潜在市场规模和发展特点。如果挑选了一个不适合的领域或方向，客户群体本身都面临较大的经营压力,怎么可能在IT建设上投入很多成本？\n- 分析市场结构与竞争格局\n- 细分并定位目标客群\n- B端产品竞品分析\n通过调研深入理解客户业务\n- 业务调研的五个环节\n- 业务调研的三个阶段\nB端业务调研的目的和业务分析框架\n- 业务调研的三个目的\n- 业务分析的三层框架\n开展新业务时，首先要结合自身的【战略定位】、【组织发展阶段】，思考业务目标和当前情况，从而推导出基本的【管控模式】；\nB端业务分析框架\n- 战略层：审视企业的现在和未来\n- 战术层：设计承载战略落地的打法\n- 执行层：定义业务运作的规则体系\nB端业务的多利益方识别与管理\nB端业务涉及多角色协同，并且不同角色的诉求不同，在产品设计前，首先应梳理清楚业务中存在哪些角色和利益方，这是理解业务运作、合理安排调研计划的基本前提。\n利益方的另一个名字叫作涉众，也可以叫干系人。

# 一句话定义
BPMN 事件建模中最常见的五种错误及纠正方法，覆盖缺少事件、命名不当、重名、终止事件误用、消息事件跨泳道误用。

# 核心内容

## 五大常见错误速查
| 错误 | 正确做法 |
|:---|:---|
| 缺少开始/结束事件 | 每个流程和子流程都应有开始和结束事件 |
| 事件命名不充分（"流程开始"）| 用业务语义命名；无特定触发器时可省略 |
| 事件重名（多个不同事件用同一名称）| 区分命名（如"订单提交" vs "退款申请"）|
| 混淆结束/终止事件 | 终止=一路结束全停，结束=各路独立完成 |
| 跨泳道使用消息事件（池内道间用消息捕获/抛出）| 池之间用消息流，池内用顺序流 |

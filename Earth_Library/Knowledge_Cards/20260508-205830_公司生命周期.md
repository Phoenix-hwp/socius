# 一句话定义
公司依次经历项目型→产品型→平台型→生态型四种形态，成长路径可抽象为项目产品化→产品平台化→平台生态化三大转化，与生物圈（个体→种群→群落→生态）逐层对应。

# 核心内容

## 四类公司
| 类型 | 特征 | 生物圈类比 | 用户关系 |
|:---|:---|:---|:---|
| 项目型 | 抓零散单次机会 | 个体 | 个体价值 |
| 产品型 | 找到稳定盈利点 | 种群 | 个体黏性 |
| 平台型 | 有供需多方、自我增强回路 | 群落 | 群体黏性 |
| 生态型 | 行业优化效率，用户深度耦合 | 生态 | 生态黏性 |

## 三大转化
1. **项目产品化**：从提供"个体价值"到产生"个体黏性"，提升生存能力
2. **产品平台化**：出现供需多方（如老师/学生、卖家/买家），从"个体黏性"到"群体黏性"，找到自我增强回路
3. **平台生态化**：面向整个行业优化效率、重塑生态，从"群体黏性"到"生态黏性"

## 生态 vs 平台的判别标准
商户只用你获客 → 平台；商户用你管会员/供应商且供应商也依赖你 → 生态

# 使用场景
- 判断一家公司当前处于哪个阶段，推导下一步战略重心
- 评估某业务线的成长阶段，确定资源配置优先级
- 做竞品分析时，识别对手所处阶段与其战略动作是否匹配

# 与其他框架的关系
- 与 **商业模式画布**：公司生命周期告诉你在哪个阶段，画布帮你设计该阶段的商业模式
- 与 **SWOT 分析法**：阶段不同，优势/劣势/机会/威胁的相对权重也不同
- 与 **产品生命周期**：公司层面的生命周期 vs 产品层面的生命周期，视角不同但可对照

# 一条边界
本框架适合分析单业务公司或核心业务方向明确的公司。不适用于分析多元化集团（旗下多条业务线各自处于不同阶段），此时应逐业务线分别套用。

# 一句话定义
业务架构代表整体的多维业务视图（能力/端对端价值交付/信息/组织），是企业架构的龙头——首先要确保业务架构可靠，再以它为指导逐步分析数据、应用和技术架构。

# 核心内容

## 业务架构要素
| 层级 | 要素 |
|:---|:---|
| 4 个核心要素 | 价值流、能力、组织、信息 |
| 6 个扩展要素 | 利益相关者、战略、政策、产品、指标、举措 |

## 两大主流业务架构路线

### 路线一：波特价值链 + 流程（PCF + BPMN）
- **波特价值链**：企业创造价值的过程可分解为一系列互相关联的增值活动（基础活动 5 类 + 支持性活动 4 类），总和构成价值链
- **流程分类框架（PCF）**：APQC 提出，分 5 个层级（流程分类 → 流程组 → 流程 → 活动 → 任务），提供跨行业通用分类法
- **BPMN**：以标准化符号对业务流程进行可视建模

### 路线二：价值流 + 能力（BA Guild / TOG）
- **价值流**：为客户创建结果的端对端活动集合，每个阶段都有价值增量
- **业务能力**：企业为达成特定目的所拥有或交换的特定能力和产能

## 业务架构的作用
- 企业整体业务能力建设的基础
- 业务战略到业务执行的桥梁
- 统一业务语言、驾驭复杂性
- 牵引 IT 架构（数据/应用/技术）
- 减少 IT 重复投资

# 使用场景
- 启动企业级数字化转型前的顶层规划
- 业务与 IT 部门对齐时作为共同语言
- 重大组织变革前做业务能力盘点

# 与其他框架的关系
- 与 **企业架构**：业务架构是企业架构的子集（企业架构 = 业务架构 + IT 架构），业务架构是企业架构的龙头
- 与 **业务架构核心要素**：本卡为整体概览，核心要素卡展开"价值流+能力"路线的细节
- 与 **BPMN 系列卡**：BPMN 是业务架构中流程视图的具体建模语言

# 一条边界
业务架构不直接涉及技术选型、API 设计、数据库建模——那是 IT 架构的职责。业务架构应该保持"技术和平台无关"，只描述业务是什么、需要什么能力，不规定用什么技术来实现。

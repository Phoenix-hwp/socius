# 一句话定义
制定产品关键指标的三条硬原则（事前制定/平衡虚实/防作弊）和一条对齐基线（使命愿景价值观），确保指标真正衡量用户价值而非表面数据。

# 核心内容

## 三条制定原则

| 原则 | 核心要求 | 反模式 |
|:---|:---|:---|
| **事前制定，不可事后调整** | 项目执行前定好；无重大变化不调整；提前想清楚退出机制和终止投入条件 | 过程中发现完不成就"灵活调整"指标 |
| **不能太虚也不能太实** | 太虚难考核 → 用 SMART 原则兜底（具体/可衡量/可达到/与目标相关/有时间限制） | 太实→易作弊，用「防作弊三条」兜底 |
| **防作弊三条** | ① 非受迫（用户不被逼做无价值动作）；② 无诱导（非奖励利益驱动）；③ 成功行为（行为本身为用户创造价值，非只对公司有价值） | 考核注册数→刷注册；考核时长→自动播放 |

## 基石：必须对齐
关键指标的制定须与**业务目标**、**产品使命愿景价值观**相契合——否则指标再 SMART 也只是在错误方向上跑得更快。

# 使用场景
- 每个版本迭代规划时，为本版本定义 1-3 个关键指标及对应的退出条件
- 评审运营提出的 KPI 时，用防作弊三条快速识别指标是否会被刷
- 向管理层汇报时，展示指标与产品生命周期阶段的匹配关系

# 与其他框架的关系
- 与**产品生命周期**：本卡定义每阶段的衡量标准；生命周期卡定义各阶段特征与运营重心
- 与**KANO 模型**：KANO 功能分类随生命周期演变，指标也随之变化 — 验证期关注基础功能覆盖率，爆发期关注亮点功能驱动的转化
- 与**需求分析十三要素五步法**：步骤二的发生频率和步骤八的实际价值，为指标选择提供输入

# 一条边界
本卡只提供指标制定的原则框架，不提供各阶段的具体指标清单。实际操作需结合产品生命周期阶段、业务类型（交易型/内容型/SaaS型）自行定义。另外，防作弊三条是必要条件非充分条件 — 通过三条检查不代表指标就是好指标。

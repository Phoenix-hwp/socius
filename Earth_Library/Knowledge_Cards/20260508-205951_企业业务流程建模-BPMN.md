# 一句话定义
BPMN 五大元素类别（流对象/数据/连接对象/泳道/工件）的完整符号总览，涵盖事件的三阶段、活动的三层级、网关的四类型及各元素变体。

# 核心内容

## 活动三层级
| 层级 | 特征 |
|:---|:---|
| 任务（Task）| 原子性活动，不可打断 |
| 子流程（Sub-Process）| 内部可用活动/事件/网关建模 |
| 调用活动（Call Activity）| 引用全局流程或全局任务，需精确匹配输入输出 |

## 关键区分
- **Manual Task vs User Task**：Manual Task 不受 BPM 引擎管理，User Task 受引擎追踪
- **子流程 vs 调用活动**：子流程是"嵌入定义"，调用活动是"外部引用"

## 四大网关
| 网关 | 规则 |
|:---|:---|
| 排他网关 | 只有一条路径被选择 |
| 并行网关 | 所有路径同时选择 |
| 包容网关 | 可执行多条，可设条件 |
| 事件网关 | 由中间捕获事件驱动，先到先触发 |

## 数据层
BPMN 本身不提供内置数据模型或查询语言，默认用 XML Schema + XPath，工具供应商可替换。

# 使用场景
- BPMN 初学者建立元素全貌认知
- 实际建模时快速查阅标准符号
- 作为 BPMN 指南系列的索引地图

# 与其他框架的关系
- 与 **BPMN 2.0 标准**：标准卡为规范全貌，本卡为元素符号图解
- 与 **BPMN 入门到掌握**：入门卡为快速版，本卡为详细元素说明

# 一条边界
BPMN 不提供内置数据结构模型或查询语言。如果需要在流程中做复杂数据转换和验证，需配合其他工具（如 DMN 决策规则）。

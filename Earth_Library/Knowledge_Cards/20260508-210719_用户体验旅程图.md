# 一句话定义
从用户视角出发，以叙述故事方式可视化用户使用产品的全流程体验（行为→情绪→痛点→机会点），是发现设计机会点的核心工具。

# 核心内容

## 绘制六步法

| 步骤 | 核心动作 | 注意事项 |
|:---|:---|:---|
| 1. 创建用户画像 | 基于真实用户信息（基本信息/使用习惯/消费行为/场景/目的）形成虚拟人物原型 | 确保旅程图的真实性 |
| 2. 分析行为路径 | 确定任务起点→任务目标→衍生子场景→形成一/二级行为路径 | 用户行为未必按序进行，需用流程图连接复杂场景 |
| 3. 拆分场景 | 按【所做所想】【情绪】【痛点】【用户期望】【设计机会点】归类 | 以产出完整流程为中心，勿过早沉浸细节 |
| 4. 设定情绪曲线 | 每个节点标注积极/平静/消极三种情绪；高情绪触点→优化空间；低情绪触点→解决问题 | 将体验可视化，直观看到情绪波动 |
| 5. 讨论解决方案 | 针对痛点产方案：功能扩展/交互优化/视觉优化/性能优化，可对标竞品 | 聚焦痛点对应的具体解决方向 |
| 6. 产出旅程图+排期表 | 汇总为可阅读旅程图→同步给设计/产品团队→评估落地时间/执行成本→推动优化 | 回推排期的闭环 |

# 使用场景
- 新产品设计阶段，梳理目标用户在核心场景中的完整体验路径
- 现有产品体验优化，定位用户流失/不满的具体环节
- 跨团队对齐（产品/设计/研发/运营），作为共享的用户体验参照物

# 与其他框架的关系
- 与**同理心地图**：旅程图还原操作路径+情绪；同理心地图站用户视角发现问题但不还原路径——两者互补
- 与**如何绘制用户旅程图**：本卡是概念+方法论概述，后者提供九步标准操作流程+常见误区
- 与**需求分析十三要素五步法**：旅程图中的痛点可输入十三要素步骤二「了解基本场景」中

# 一条边界
旅程图依赖真实用户数据支撑——仅凭假设构建会偏离现实。不适合「猜测型」快速判断，需要先完成用户调研。另外，旅程图是动态文档，需随产品迭代和用户反馈持续更新。

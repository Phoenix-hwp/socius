# 一句话定义
BPMN 网关控制流程中序列流的发散和收敛，决定路径的分支、分叉、合并和连接，包含独占/包含/并行/事件/复杂五种类型，是流程"道路上的分流点和汇合点"。

# 核心内容

## 五种网关速查
| 类型 | 发散行为 | 收敛行为 |
|:---|:---|:---|
| 独占网关 | 仅取一条路径 | 合并多条路径（任一路到达即通过）|
| 包含网关 | 可同时走多条路径 | 需等待所有已激活路径到达 |
| 并行网关 | 所有路径同时执行 | 需等待所有路径到达 |
| 事件网关 | 由先发生的事件决定路径 | — |
| 复杂网关 | 由表达式精确定义 | 由表达式精确定义 |

## 六种路径逻辑
| 逻辑 | 英文 | 作用 |
|:---|:---|:---|
| 分叉（Fork）| AND-Split | 一路分多路并行 |
| 连接（Join）| AND-Join | 多路并一路 |
| 排他（Exclusive）| XOR-Split | 条件分支，仅取其一 |
| 基于事件（Event-Based）| — | 由事件（消息/时间）决定取哪条路 |
| 包含（Inclusive）| OR-Split | 一组独立的是/否决策，至少取一条 |
| 合并（Merging）| OR-Join | 将多条排他路径合并为一条 |

## 关键区分
- **排他 vs 包含**：排他=只能走一条（互斥）；包含=可走多条（独立条件各自评估）
- **并行 vs 包含**：并行=全部必走；包含=各自独立判断走不走
- **事件网关**：决定路径的不是流程数据而是外部事件（如先收到哪个消息）

# 使用场景
- 流程中有条件分支时选择独占或包含网关
- 流程需要并行处理多个任务时使用并行网关
- 多个外部消息可能到达，以先到者为准时用事件网关

# 与其他框架的关系
- 与 **BPMN 元素总览**：网关是五大元素类别之一
- 与 **BPMN 指南(2)活动**：网关控制"活动之间的走向"，活动是"走的过程中做什么"
- 与 **BPMN 指南(10)连接**：网关控制路径逻辑，连接表达各元素间的连线类型

# 一条边界
分叉不一定非要用并行网关——使用活动的多个传出序列流（"不受控制流"）是更常见、更简洁的做法。并行网关通常与其他网关组合使用才出现。不要滥用网关符号。

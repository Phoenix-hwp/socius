# Summary
产品起步的逻辑的核心要点归纳。

# Details
产品起步的逻辑顺次可以归纳为：【习惯打造】（简称为：“习惯”模块）、【启程】、【发现】、【精通】。\n- 习惯\n这要求我们先打造出某个对用户有价值的闭环，这一模块应该做到让用户来了可以获得价值并且下一次还愿意来，甚至愿意推荐朋友来。这个最小的产品模块已经可以用来做“留存假设”的验证。\n- 启程\n启程，即用户的第一次体验。产品的早期使用者往往是高手行家，我们也长把这群人称作种子用户、天使用户。他们即便没人手把手指导，也能顺利上手。所以，启程模块是产品检验的对象扩展开以后，做给新手用户的，最常见的就是各种产品里的“新手上路”模块。\n- 发现\n有了一批新人用户之后，我们算是验证完了“启程”与“习惯”模块。这时候产品就应该进入推广阶段，开始做“发现”模块。我们要发觉出用户在何时、何地会对产品产生第一印象，会通过什么渠道第一次接触产品。比如：对于一个手机App来说，从用户在应用市场里看到广告、搜索产品名称、下载安装，直到第一次打开App之前的整个过程都是“发现”模块。\n- 精通\n产品运营了一段时间后，会有相当数量的用户对产品了如指掌，这时候我们就有必要为他们打造“精通”模块，让他们不断接受到新的刺激。“精通”模块属于高级功能，我们可以考虑让高级用户参与其中，充分利用最热情用户的深层次需求和驱动力。比如：一个论坛让高级会员成为版主，一款游戏让资深玩家做分区的督导者。\n用户使用产品的逻辑依次经历四个模块：发现、启程、习惯、精通，但做产品起步的逻辑顺序则是：习惯、启程、发现、精通。\n这一产品起步方法更适用与从少量用户逐步扩展到大量用户的产品，但如果服务的是少数大客户，第一次交付时的产品模块就要相对完整。\n用户运营四象限图
\n在用户运营上，可以把用户按照【对产品的熟悉度】和【对产品的忠诚度】分为四个象限。\n- 低熟悉度低忠诚度\n这类用户属于非目标用户，在这些人身上花费时间精力的投入产出比最低，特别是当我们需要付出金钱去“买”用户的时候，在这类用户身上的钱可以说是白花的，它仅仅对短期的“虚荣指标”有贡献。因此，对这类用户保持“广播式”的覆盖即可，后续慢慢影响。\n- 低熟悉度高忠诚度\n这类用户多见于新用户群体，做用户运营时，要做好“启程”模块，给他们一套新手上路和养成的办法，提升留存和转化。比如：社交产品里让用户快速加到好友，以及游戏里的新手村任务等。\n- 高熟悉度低忠诚度\n这类用户属于老用户，需要使用各种激活召回的手段。比如电商中发优惠券，社群组织资深用户线下聚会\n- 高熟悉度高忠诚度\n这类属于明星用户或者种子用户，要做好维护，用授权的办法让他们能更多地帮到产品，这对应着“精通”模块。比如让群用户来组织活动，给他们一些产品内的特权，如论坛的内容审核、版块管理权等。

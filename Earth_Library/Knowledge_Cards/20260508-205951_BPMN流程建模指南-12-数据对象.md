# 一句话定义
BPMN 数据元素（数据对象/数据存储/数据输入/数据输出）提供"活动需要什么信息"和"活动产生什么信息"的可视化标注，是连接业务流程与信息流转的关键桥梁。

# 核心内容

## 四种数据元素速查
| 元素 | 符号 | 含义 | 生命周期 |
|:---|:---|:---|:---|
| 数据对象 | 右上角折角的文档图标 | 流程中创建和使用的信息 | 有明确定义的生命周期和访问限制 |
| 数据存储 | 圆柱体图标 | 流程范围外持久存储的信息 | 活动可检索或更新 |
| 数据输入 | 空心箭头指向活动 | 声明使用特定类型数据作为输入 | 可有多个 |
| 数据输出 | 实心箭头离开活动 | 表示将输出特定类型的数据 | 可有多个 |

## 数据在 BPMN 中的位置
- BPMN 本身不提供内置数据结构模型或查询语言
- 默认使用 XML Schema + XPath
- 工具供应商可以替换为其他数据建模方案

# 使用场景
- 明确流程中每个活动的输入和输出数据
- 标注哪些数据在流程结束后需要持久化到数据存储
- 用数据对象表示流程状态或流程产生的文档

# 与其他框架的关系
- 与 **BPMN 元素总览**：数据是五大元素类别之一
- 与 **BPMN 指南(10)连接 - 数据关联**：数据关联用于在数据对象与活动的输入/输出之间移动数据
- 与 **UML 类图**：BPMN 描述流程中的数据流转，UML 类图描述数据的结构定义——两者互补

# 一条边界
数据对象描述的是"什么数据在流转"，不是"数据结构长什么样"（那是 ER 图或 UML 类图的职责）。不要在 BPMN 数据对象中过度详述字段级细节，保持流程层面的抽象。

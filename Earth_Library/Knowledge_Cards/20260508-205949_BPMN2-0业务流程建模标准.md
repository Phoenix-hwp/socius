# 一句话定义
BPMN（Business Process Model and Notation）是 OMG 发布的最广泛使用的业务流程建模标准，提供从流程编排、编排到协作的完整符号集，ISO 对等标准为 ISO/IEC 19510:2013。

# 核心内容

## 三类流程
| 类型 | 核心含义 | 应用场景 |
|:---|:---|:---|
| 流程编排 | 单一组织内部或对外暴露的工作流 | 内部流程文档、系统可执行流程 |
| 编排 | 关注多参与者的消息交互顺序 | 跨组织业务流程协议 |
| 协作 | 用池和消息流建模多方交互 | B2B 对接、系统集成 |

## 四类基础元素
| 类别 | 包含 | 作用 |
|:---|:---|:---|
| 流对象 | 事件、活动、网关 | 定义流程的主干逻辑 |
| 数据 | 数据对象、输入/输出、存储 | 建模流程中数据的创建和流转 |
| 连接对象 | 顺序流、消息流、关联 | 连接元素，表达执行顺序和通信 |
| 泳道 | 池、道 | 按参与者/角色对活动分组 |

## 关键优势与局限
- **优势**：符号集最丰富、ISO 正式标准、美国政府和国防部大量采用
- **局限**：需要培训才能用全套符号，不同工具支持子集不同，多层级关系不明显

# 使用场景
- 企业内部流程梳理和文档化
- 跨系统集成时定义接口协议
- 流程自动化前的规范建模

# 与其他框架的关系
- 与 **UML 流程图/状态图/类图**：BPMN 侧重业务流程，UML 侧重软件设计；互补而非替代
- 与 **业务架构/企业架构**：BPMN 是业务架构中流程视图的具体建模语言

# 一条边界
BPMN 不是数据建模工具（ERD 更合适），也不是 UI 设计工具。"怎么画流程"用 BPMN，"为什么这么画"和"用户路径"需要产品方法和用户研究补充。

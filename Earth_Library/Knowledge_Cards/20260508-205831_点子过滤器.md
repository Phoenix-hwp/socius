# 一句话定义
点子过滤器是产品概念筛选框架，通过内部（能力+意愿）和外部（价值+成本）四个维度，在早期判断一个产品点子是否值得投入，避免资源浪费。

# 核心内容

## 筛选框架
| 维度 | 组件 | 核心问题 |
|:---|:---|:---|
| **内部·能力** | 人/财/物 | 团队有无技术高手？资金是否充足？有无数据积累？ |
| **内部·意愿** | 使命/愿景/价值观 | 这事符合公司的长期目标吗？ |
| **外部·价值** | 行业天花板 + 用户价值 | 赛道够宽吗？用户价值高吗？ |
| **外部·成本** | 宏观环境 + 微观行业 | PESTEL 各维度是否有利？行业玩家、上下游、进入者、替代者情况如何？ |

## 核心洞察
- **"势"是关键**：一个点子只有顺势而为才容易通过过滤器。"势"包括行业浪潮、公司基因（宏观）和用户特质、需求特性、场景特点（微观）
- **深入定制是双刃剑**：对某类用户/需求的深度定制，既可能成为竞争壁垒，也可能变成产品的牢笼

# 使用场景
- 产品立项前，对多个候选点子做快速筛选
- 创业者在冲动投入前做理性自查
- 产品经理接手新需求时，判断该需求是否值得排入迭代

# 与其他框架的关系
- 与 **四大宏观环境分析模型**：外部·成本维度中直接用 PESTEL 分析宏观大环境
- 与 **SWOT 分析法**：点子过滤器是"要不要做"的早期判断，SWOT 是"决定了做"之后的态势分析
- 与 **精益画布 & PMF**：点子过滤器回答"值不值得做"，精益画布回答"怎么做"，PMF 回答"做对了没有"

# 一条边界
点子过滤器适合**早期、有多个可选方向**的场景。如果已经进入执行阶段且投入了一定资源，应转向 SWOT 分析和商业模式画布来做态势评估和策略设计，而非回头用过滤器重新讨论"值不值得"。

# 一句话
覆盖"B 端产品和 C 端有什么不同、怎么做 B 端产品从调研到方案到迭代的完整闭环"，是与产品方法论域高度交叉的专业域。

# 领域地图

## 场景 → 工具速查
| 你想做什么 | 用哪张卡 | 一句话说明 |
|---|---|---|
| 理解 B 端产品的发展阶段和路径 | B端产品的演进路线 | 定制化→商业化→多元化→平台化四阶段演进，不是所有产品都走完四阶段 |

| 管理 B 端产品的需求池 | B端产品需求管理与迭代 | 三层需求分类（业务/用户/技术）× 四类价值（商业/业务/用户/技术）映射框架 |

## 常见组合路径
- **路径1**：做一个 B 端产品从立项到规划 → 先读 B端产品市场分析与业务调研（调研）→ 再读 B端产品的演进路线（定位发展阶段）→ 用 B端整体方案设计做方案规划 → 最后参考 B端产品需求管理与迭代建立需求管理机制
- **路径2**：判断 B 端产品该往哪个方向演进 → 先读 B端产品的演进路线确认当前阶段 → 交叉到《商业战略与建模》域的 SWOT / 商业模式画布做战略判断 → 再回到 B端整体方案设计更新方案
- **路径3**：B 端产品需求优先级排序 → 先读 B端产品需求管理与迭代做需求分类和价值映射 → 交叉到《产品方法论》域的 KANO 模型做功能优先级 → 若涉及用户研究，交叉到《用户研究》域
- **路径4**：从市场分析驱动产品方案 → 先读 B端产品市场分析与业务调研做完备调研 → 用调研结论驱动 B端整体方案设计 → 后续用 B端产品需求管理与迭代管理持续输入

## 域内缺口
- **不含 B 端 SaaS 定价模型**：不覆盖按座位、按用量、按功能模块等 B 端定价策略
- **不含 B 端销售和实施方法论**：不覆盖售前演示、POC、实施交付、客户成功等
- **不含 B 端系统集成 / API 设计**：不覆盖 ESB、API 网关、数据集成等技术实现
- **不含 B 端竞品分析方法**：竞品生态卡在零散域暂不归入；Agent 可用 B端市场分析与业务调研 + 交叉《商业战略与建模》域 SWOT / 波特五力做竞品分析

## 交叉域参考
- 涉及"B 端产品 vs 一般产品方法论"时，先读《产品方法论》域概览卡（通用方法）→ 再回本域（B 端特殊场景）
- 涉及"B 端商业模式设计"时，参考《商业战略与建模》域概览卡（商业模式画布 / 精益画布），再结合本域的演进路线
- 涉及"B 端业务流程建模"时，参考《流程建模与架构》域概览卡，用 BPMN 画业务流程图
- 涉及"B 端用户（多角色）怎么研究"时，参考《用户研究》域概览卡，叠加本域的多利益方识别方法

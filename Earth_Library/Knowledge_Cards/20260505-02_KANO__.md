# Summary
KANO模型的核心要点归纳。

# Details
基础功能\n基础功能是指当这类功能没有实现时，用户对产品是“极其不满”的。但是，即使这类功能做的再好，用户也认为是“理所应当”。\n随着功能的完善（横轴数值的增加），用户满意度（对应的纵轴数值）虽然有所增长，但始终没有突破0，最多就是趋近于0。\n基础功能是必须要实现的功能点，但它的实现无法给用户带来很高的满意度，只能消除用户的不满。\n如果一时无法确定基础功能，可以通过提问用户如下问题来获知：\n- 如果产品没有功能A，你觉得如何？请从很满意、一般满意、无所谓、不太满意、很不满意（或满意程度5~1）这五个选项中选出一个\n- 如果产品有了功能A，你觉得如何？同上给出相应的答案。\n亮点功能\n亮点功能是指当没有这个功能是，用户并不会不满意或觉得有问题，但是一旦有了这个功能，用户就会大喜过望，甚至“赞不绝口”。这是因为这些功能提供了用户意料之外的“妙用”。\n随着功能的完善（横轴数值增加），用户满意度（对应的纵轴数值）从0开始正相关地增长。\n亮点功能是用户忠诚度、口碑传播的基础。\n亮点的常见特性是：用户没见过、未经市场检验、如果被认可就能获得巨大回报。\n对亮点的挖掘主要靠的是产品人对用户人性和心智的理解，因为亮点是用户想不到、说不出的东西，必须深刻洞察用户的心理。\n亮点功能的选择\n亮点功能的选择要根据公司的转态和产品的阶段来区分看待。大公司和小公司，成熟产品和早期产品，对亮点的打造是不同的。\n- 小公司和早期产品，应该优选成本低的亮点，仅将其作为锦上添花的功能。\n- 大公司和成熟产品，可以大胆尝试一些让自己保持领先地位的亮点，以寻求更大突破。\n期望功能\n期望功能在KANO模型中是一条比较平直的线条。\n期望功能的特点是：少了他用户有点不满意，但也不是无法接受；多了它用户会觉得不错，但也不是欣喜若狂。它对产品而言往往是“多多益善”。对这类产品功能的选择方式也比较简单——先做性价比高的。\n随着功能的完善（横轴数值变大），用户满意度（对应的纵轴数值）从负数开始渐渐增长，然后穿过坐标系原点，变为正数后继续增长。功能实现程度和用户满意度正比例相关。\n之所以取名为“期望功能”，是因为这类功能都是用户的期望。\n产品具备了期望功能，用户就会继续使用，但又因为缺乏惊喜感，用户不大可能主动去传播，这样的产品很难取得成功。如果只是简单地通过和用户交流来采集需求，最终实现的可能只是期望功能。\n无差别功能\n无差别工能指的是做与不做，用户对产品的感受都没有变化。\n对待这样的功能，可以采用低成本原型验证方法进行验证。当我们准备对任何类型的功能做取舍是，都可以先做低成本验证。\n反向功能\n返现功能表示：做得越多，用户越讨厌。\n随着功能的完善（横轴数值的增加），用户满意度（对应的纵轴数值）下降，并且穿过图中的原点，这意味着用户满意度已经下降到了我们对产品定位是所做预期之下。\n但这类的功能又不能简单的理解为直接放弃不做。一个产品的用户是多种多样的，不同用户的目标也各不相同。所以某个功能对于某种用户来说可能是反向的，但对于另一些用户来说却可能是正向的。比如：百度的广告，对普通搜索用户来说，搜索结果里广告越多，满意度就越差，但对投放广告的用户来说，又希望搜索结果中有自己的广告。

# 一句话定义
BPMN 中间事件发生在开始和结束之间，可出现在正常流程中（延迟/抛出/捕获）或附加在活动边界上（中断/非中断地触发异常流），共覆盖消息/时间/升级/补偿/条件/链接/信号/错误/取消/多重/并行多重 11 种类型。

# 核心内容

## 两种使用位置
| 位置 | 行为 | 关键区分 |
|:---|:---|:---|
| 正常流程中 | 令牌到达时触发（延迟或抛出）| 可捕获或抛出 |
| 附加在活动边界上 | 被触发时将正常流变为异常流 | 仅捕获；实线边界=中断活动，虚线边界=不中断 |

## 关键边界事件速查
| 事件 | 中断/非中断 | 关键规则 |
|:---|:---|:---|
| 消息 | 两种都有 | 实心边界=中断，虚线边界=不中断 |
| 时间 | 两种都有 | 同上。常用场景：超时后触发备用路径 |
| 升级 | 默认不中断 | 可覆盖为中断（实线边界）|
| **错误** | **仅中断** | 无虚线版本。对命名错误或任何错误反应 |
| **取消** | **仅中断** | 仅用于事务子流程 |
| 补偿 | 仅不中断 | 补偿只能在活动完成后触发 |
| 条件 | 两种都有 | 条件变为真时触发 |
| 信号 | 两种都有 | 比错误事件范围更大（跨进程）|

## 特殊事件
- **链接事件**：连接一个流程的两部分，仅限于单个流程级别，成对使用（源可有多个，目标只能一个），可用作"页外连接器"
- **平行多重事件**：所有分配的触发器都需要，不同于普通多重事件（仅需任一）

# 使用场景
- 流程中需要"等待某个条件"时选择对应类型的中间捕获事件
- 活动需要超时处理时附加时间边界事件（虚线=预警，实线=中断）
- 需要事务回滚时使用取消和补偿事件组合

# 与其他框架的关系
- 与 **BPMN 指南(4)开始事件、(5)结束事件**：事件三章各自覆盖一个阶段
- 与 **BPMN 指南(7)事件使用规则**：通用规则覆盖所有事件，本卡为中间事件专项
- 与 **BPMN 指南(8)事件常见错误**：错误 5"跨泳道消息事件"直接关联中间事件的边界用法

# 一条边界
中间事件不能触发流程的开始或结束——那是开始/结束事件的职责。附加边界的事件必须"捕获"而非"抛出"（抛出只能在正常流程中）。错误事件只有中断版本，不要试图用虚线边界表示"非中断错误"。

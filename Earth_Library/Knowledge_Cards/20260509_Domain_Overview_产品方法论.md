# 一句话
覆盖"怎么做产品决策、怎么排功能优先级、怎么判断方向、怎么从 0 到 1、怎么评价好坏"，是日常产品工作最常命中的域。

# 领域地图

## 场景 → 工具速查
| 你想做什么 | 用哪张卡 | 一句话说明 |
|---|---|---|
| 系统化拆解一个问题 | 5W2H分析法 | What/Why/Who/When/Where/How/HowMuch 七维拆解，用户侧和设计侧双向交叉审视 |
| 给功能排优先级 | KANO模型 | 按实现程度与满意度关系分五类（基础/亮点/期望/无差别/反向），资源有限时做取舍 |
| 做需求分析，从模糊需求到产品方案 | 需求分析的十三要素五步法 | 5 步覆盖 13 要素：角色→场景→动机→替代方案→产品设计，层层递进找最优解 |
| 判断产品处于哪个阶段，该关注什么 | 产品生命周期 | 验证期→爆发期→平台期→衰退期四阶段，每阶段对应不同产品/运营/商业策略 |
| 产品从 0 到 1 怎么起步 | 产品起步的逻辑 | 习惯→启程→发现→精通四模块，关键洞见：用户使用逻辑与产品构建逻辑顺序相反 |
| 从单一产品演进到产品矩阵 | 从产品到产品矩阵 | PSF→PMF→PRF 三阶段验证链，矩阵中每个产品须满足「可复用、能积累、善生死」 |
| 制定每个阶段该看什么指标 | 产品各阶段的关键指标 | 三条硬原则（事前制定/平衡虚实/防作弊）+ 一条基线（使命愿景价值观）|
| 判断一个产品好不好 | 好产品的评价标准 | 静态（有用→好用→爱用）和动态（个体→群体→生态黏性）双维度评判 |
| 评价产品设计好不好 | 如何理性评价产品设计的好坏 | 八维度（企业/用户/产品/交互/视觉/开发/运营/社会责任）结构化评审 |
| 理解产品从实体到服务的演变 | 产品服务系统 | 实体导向→使用导向→结果导向三类 PSS，解释交付形态的演变逻辑 |
| 做 MVP / 产品创新验证 | 5MVVP框架 | 五轮递进冲刺（Paperwork→Prototype→Product→Promotion→Portfolio），逐轮验证假设、收紧资源 |
| 理解产品思维的核心命题 | 产品思维 | 顶层分类节点，聚合 24 篇方法论笔记，核心命题：什么是好产品 |
| 做竞品分析 | 竞品生态 | 问题×方案四象限矩阵，突破窄视角发现跨品类竞品和潜在颠覆者 |

## 常见组合路径
- **路径1**：做一个新产品（从 0 到 1）→ 先用 5W2H 拆解问题 → 用需求分析十三要素做需求梳理 → 用 KANO 排功能优先级 → 用 5MVVP 框架做验证冲刺 → 用产品起步的逻辑指导上线节奏
- **路径2**：做产品评审 / 方案评估 → 先用 5W2H 理解方案全貌 → 参考好产品的评价标准判断方向 → 用如何理性评价产品设计做结构化评审 → 用 KANO 检查功能取舍
- **路径3**：产品进入增长期，不知道看什么数据 → 先读产品生命周期确认阶段 → 再读产品各阶段的关键指标制定衡量标准 → 如果涉及产品扩展，读从产品到产品矩阵
- **路径4**：做产品战略复盘 → 先读产品生命周期定位阶段 → 再读好产品的评价标准做质量判断 → 用产品服务系统思考交付形态 → 用从产品到产品矩阵规划下一步
- **路径5**：做竞品分析 → 先读竞品生态打开四象限视野 → 用 SWOT 做单竞品态势 → 用 KANO 对比功能优先级 → 用需求分析梳理差异背后的用户动机

## 域内缺口
- **不含 A/B 测试方法论**：本域有指标制定原则，但具体 A/B 测试设计、统计显著性判断不覆盖
- **不含用户增长（Growth Hacking）模型**：海盗指标（AARRR）、增长飞轮等不覆盖
- **不含定价策略**：不覆盖付费墙、价格弹性、捆绑销售等定价方法
- **不含技术架构 / 系统设计**：本域偏产品决策，技术实现不覆盖
- **竞品分析**：已有竞品生态卡覆盖四象限分析框架，但传统竞品功能对比表的制作规范不覆盖

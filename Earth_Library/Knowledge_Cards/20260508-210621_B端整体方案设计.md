# 一句话定义
B端整体方案设计的六步法：从核心业务流程切入→定义问题边界→明确产品定位→梳理应用架构→定义场景与功能模块→规划演进蓝图，最后以数字化五阶段收官。

# 核心内容

## 六步设计流程

| 步骤 | 核心动作 | 关键注意事项 |
|:---|:---|:---|
| 1. 确定核心业务流程 | 梳理业务主干脉络：参与方+涉及系统；只画关键场景和节点不陷细节 | 只有定义清业务边界才能定义产品边界 |
| 2. 定义问题边界 | 明确产品解决「哪些场景下」的「什么具体问题」 | — |
| 3. 明确产品定位 | 宏观层：目标客户→解决什么问题→提供什么价值。执行层：由几套子系统组成→各自的目标用户和问题 | 子系统是在应用层面划分，技术实现上可能统一底层+多套前端 |
| 4. 梳理应用架构 | 新系统如何与公司现有架构融合；商业化产品如何与甲方架构融合；多产品协同需一体化架构 | 工具型B端产品（工业软件/绘图等）不用过多考虑应用架构 |
| 5. 定义场景与功能模块 | 做加法→场景→功能模块规划蓝图=系统骨架。三种抽象思路：基于业务领域/基于业务场景/基于业务对象 | 常见问题：模块层次混乱、后期随意摆放新功能 |
| 6. 规划演进蓝图 | 按业务优先级分期：①核心痛点(最小闭环)→②特殊刚需→③风控/运营强化 | 业务本质不变则功能模块不应结构性改动 |

## 三种模块抽象思路

| 思路 | 特征 | 典型适用 |
|:---|:---|:---|
| 基于业务领域 | 高内聚、松耦合；与底层架构设计贴合 | 最常用 |
| 基于业务场景 | 从用户体验和业务逻辑出发；场景菜单下可融合多模块 | WMS仓储（运输/进仓/出仓/退货） |
| 基于业务对象 | 以关键对象（客户/商机/订单）定义模块 | SFA CRM |

## 数字化赋能业务五阶段（严格递进）
流程化 → 线上化 → 自动化 → 数据化 → 智能化

**不可跳跃**：流程标准化未做到位→不可能线上化→无法沉淀数据→不可能智能化。

# 使用场景
- B端产品从零设计时，按六步法逐项推进确保不遗漏关键环节
- 接手已有B端产品时，对照六步法诊断产品定位/架构/模块是否有系统性缺陷
- 团队对产品方向有分歧时，用六步框架对齐各角色预期

# 与其他框架的关系
- 与**B端产品演进路线**：整体方案设计中的数字化五阶段与B端产品四阶段演进可叠加使用
- 与**B端市场分析与业务调研**：方案设计前的调研输入由此卡方法论支撑——先调研再设计方案
- 与**业务架构**：步骤一的「核心业务流程」是业务架构的价值流分析在B端产品设计中的落地

# 一条边界
六步法适用于业务型/交易型B端产品。对于纯工具型B端产品（工业软件/绘图/安全监控等），不用过多考虑应用架构（第4步）和子系统划分（第3步执行层）——核心聚焦于场景和功能的直接映射。

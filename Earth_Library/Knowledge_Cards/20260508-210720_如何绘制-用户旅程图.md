# 一句话定义
用户旅程图是产品经理的必备硬技能，通过九步标准流程从用户视角追踪完整体验路径，识别挫折点和喜悦点，为客户体验提供完整视图。

# 核心内容

## 七项关键要素

| 要素 | 说明 | 示例（电商买音箱） |
|:---|:---|:---|
| 用户画像 | 代表目标用户群体的虚构人物，含背景/需求/偏好/目标 | — |
| 旅程阶段 | 依情景划分；通常包括认知→考虑→购买→使用→忠诚→离开 | 发现产品→试用→购买→使用→寻求技术支持 |
| 触点 | 用户与产品/服务互动的所有点 | 网站/App/客服/社交媒体/实体店 |
| 情感与感受 | 各阶段情感反应 | 满意/困惑/失望/喜悦 |
| 用户行为 | 各触点的具体行为 | 搜索信息/比较产品/下单购买 |
| 痛点与机会 | 识别问题（痛点）和积极体验（机会） | — |
| 反馈与想法 | 整合用户直接反馈和间接数据 | — |

## 九步操作流程
1. 明确目标和受众
2. 收集数据和研究（调研/访谈/问卷/数据分析）
3. 确定触点和阶段
4. 绘制草图（ProcessOn 等工具；左→右时间轴；触点和阶段作纵列/泳道）
5. 描述用户情感和体验（文字/表情/颜色编码表示情感变化）
6. 识别痛点和机会点
7. 分享和讨论（跨部门协作）
8. 迭代优化（动态文档，随市场与用户变化更新）
9. 制定行动计划（针对痛点落实改进措施）

## 九大常见误区
| # | 误区 | 规避方式 |
|:---|:---|:---|
| 1 | 过度简化/复杂化 | 根据目的和受众调整详略 |
| 2 | 缺乏真实数据支持（纯假设） | 基于调研/访谈/数据构建 |
| 3 | 忽视用户情感变化 | 详细记录各阶段情绪波动 |
| 4 | 一刀切的用户画像 | 绘制多条针对不同用户群体的旅程图 |
| 5 | 内部视角过重 | 站在用户角度，非企业理想化流程 |
| 6 | 忽略触点间连贯性 | 确保触点间平滑无缝过渡 |
| 7 | 静态处理 | 定期回顾和更新 |
| 8 | 未覆盖所有关键环节（意识→售后） | 全过程覆盖 |
| 9 | 团队参与不足 | 跨部门协作制作 |

# 使用场景
- 接手新项目时，按九步标准流程从零绘制用户旅程图
- 设计评审时，用九大误区清单自检旅程图质量
- 跨部门对齐时，将旅程图作为共享参考框架驱动协作

# 与其他框架的关系
- 与**用户体验旅程图**：本卡是九步实操版，「用户体验旅程图」是六步概念版——两者互补
- 与**同理心地图**：旅程图还原操作路径+情绪；同理心地图站用户视角发现问题但不还原路径
- 与**需求分析十三要素五步法**：旅程图中的痛点可直接输入步骤三「挖掘真实动机」

# 一条边界
用户旅程图依赖真实用户数据，不适合在无用户调研的早期概念阶段硬画。此外，不同细分用户群体需要对应多条旅程图——一套旅程图覆盖所有用户是典型错误。

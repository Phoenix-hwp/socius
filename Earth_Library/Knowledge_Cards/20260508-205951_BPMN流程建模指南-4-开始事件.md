# 一句话定义
BPMN 开始事件表示流程的起点，有通用/消息/时间/条件/信号/多重/并行多重 7 种类型，分别对应不同的触发机制（手动/消息到达/定时/条件满足/信号广播/任一触发/全部触发）。

# 核心内容

## 三种事件类型
| 类型 | 位置 | 作用 |
|:---|:---|:---|
| 开始事件 | 流程起点 | 表示流程从何时/何条件启动 |
| 中间事件 | 开始与结束之间 | 影响流程但不启动或终止 |
| 结束事件 | 流程终点 | 表示流程在何处终止 |

## 七大开始事件速查
| 类型 | 触发器 | 典型场景 |
|:---|:---|:---|
| 通用 | 无预定义触发器 | 手动启动的通用流程 |
| 消息 | 参与者消息到达 | 接收到订单 → 启动订单处理流程 |
| 时间 | 特定时间/周期 | 每周一上午 9 点自动启动 |
| 条件 | 业务条件满足 | 标普 500 涨跌幅超 10% 时触发 |
| 信号 | 另一进程信号广播 | 多个流程接收同一信号同时启动 |
| 多重 | 多种方式任一种 | 消息、时间条件任满足其一即可 |
| 并行多重 | 多种方式全部触发 | 必须同时收到消息并满足时间条件才启动 |

## 关键注意
- 同一 BPMN 事件符号根据上下文含义不同：时间事件"10分钟"在顺序流中=延迟，附加在活动边界上=中断
- 条件开始事件的条件表达式不得引用流程数据上下文或实例属性（因为当时流程实例尚未创建）

# 使用场景
- 明确流程的触发方式，选择合适的开始事件类型
- 跨系统/跨组织对接时使用消息开始事件
- 定时任务/周期性流程使用时间开始事件

# 与其他框架的关系

# 一条边界
开始事件只对触发器"捕获"做出反应，不能"抛出"结果。如果需要流程启动时主动发送消息，应该用中间的"消息抛出事件"而非开始事件。
